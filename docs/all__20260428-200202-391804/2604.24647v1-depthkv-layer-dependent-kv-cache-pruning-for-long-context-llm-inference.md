## 摘要
长上下文推理是大语言模型（LLM）的一项关键能力，支持长文档理解、摘要生成和代码生成等应用。然而，高效的自回归推理依赖于键值（KV）缓存，其内存占用随序列长度线性增长，导致了主要的内存瓶颈。为了减轻这一开销，KV 缓存剪枝方法在推理过程中丢弃注意力分数较低的缓存标记（tokens）。大多数现有方法在各层之间应用统一的剪枝率，隐含地假设所有层对模型整体性能的贡献是相等的。我们证明了这一假设是次优的，因为各层对剪枝的敏感度存在显著差异。我们提出了 DepthKV，这是一个层依赖的剪枝框架，它根据各层的敏感度在层间分配固定的全局 KV 预算，而不是采用统一分配。在多个模型和任务中，DepthKV 在相同的全局剪枝率下始终优于统一剪枝，证明了通过层依赖分配可以更有效地利用 KV 缓存预算。

## Abstract
Long-context reasoning is a critical capability of large language models (LLMs), enabling applications such as long-document understanding, summarization, and code generation. However, efficient autoregressive inference relies on the key-value (KV) cache, whose memory footprint grows linearly with sequence length, leading to a major memory bottleneck. To mitigate this overhead, KV cache pruning methods discard cached tokens with low attention scores during inference. Most existing methods apply a uniform pruning ratio across layers, implicitly assuming that all layers contribute equally to overall model performance. We show that this assumption is suboptimal, as layers differ significantly in their sensitivity to pruning. We propose DepthKV, a layer-dependent pruning framework that allocates a fixed global KV budget across layers based on their sensitivity, rather than using a uniform allocation. Across multiple models and tasks, DepthKV consistently outperforms uniform pruning at the same global pruning ratio, demonstrating more effective utilization of the KV cache budget through layer-dependent allocation.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：长上下文推理（如长文档摘要、多轮对话）已成为 LLM 的核心能力，但自回归推理依赖的 **KV Cache（键值缓存）** 随序列长度线性增长，极易耗尽 GPU 显存。
*   **核心问题**：现有的 KV 缓存剪枝方法（如 H2O、SnapKV）通常在所有 Transformer 层中应用**统一的剪枝比例**。
*   **研究动机**：作者发现不同层对剪枝的敏感度显著不同，某些“关键层”在被剪枝后会导致性能剧烈下降，而统一分配预算忽略了这种层间差异，导致资源利用效率低下。

### 2. 方法论：DepthKV 框架
*   **核心思想**：打破“层层平等”的假设，根据各层的敏感度非均匀地分配固定的全局 KV 缓存预算。
*   **关键技术细节**：
    *   **层敏感度评估**：通过消融实验发现，中间层通常对剪枝更敏感，且这些层的剪枝会抑制内容生成（称为“内容放大层”）。
    *   **InfoNCE 指标引导**：引入表示学习中的 InfoNCE 损失作为层重要性的代理指标。实验证明，InfoNCE 与层敏感度高度相关，可用于在无需标注数据的情况下预测各层的重要性。
    *   **分配策略**：
        1.  **MLP（中间层保护）**：完全保留中间几层，其余层统一剪枝。
        2.  **MGA（指标引导分配）**：根据 InfoNCE 分数动态调整各层的剪枝率，给更稳健的层分配更高的剪枝率，给敏感层留出更多预算。
        3.  **MLMA（混合策略）**：结合中间层保护与指标引导分配。
*   **算法流程**：在 Prefill（预填充）阶段，计算各层的 InfoNCE 指标，根据预设的全局预算比例（如 60% 压缩率），计算各层的非均匀预算，并结合 H2O 等启发式方法筛选保留的 Token。

### 6. 主要结论与发现
*   **层间不平等性**：Transformer 各层对 KV 剪枝的敏感度存在显著且统计学上显著的差异。
*   **内容放大效应**：剪枝某些关键层（通常是中间层）会直接导致模型生成的文本变短、信息量骤减。
*   **DepthKV 的优越性**：在相同的全局内存预算下，DepthKV 的各种非均匀分配策略（尤其是 MGA 和 MLP）在所有任务上均一致优于统一剪枝。

### 7. 优点
*   **即插即用**：作为一种后处理方法，无需对模型进行微调或架构修改，易于部署。
*   **资源高效**：在不增加额外显存开销的前提下，通过优化分配逻辑提升了模型性能。
*   **深入的机理分析**：不仅提出了方法，还深入探讨了“内容放大层”的概念，为理解 LLM 内部层级功能提供了新视角。

### 8. 不足与局限
*   **非查询感知（Non-query-aware）**：剪枝决策在 Prefill 阶段完成，未考虑解码过程中 Query 的动态变化，可能在极细粒度的检索任务中丢失关键 Token。
*   **层内粒度不足**：目前仅在“层”级别进行预算分配，未考虑同一层内不同“注意力头（Heads）”的差异。
*   **超参数依赖**：虽然 InfoNCE 有效，但如何为不同模型确定最优的非均匀分配曲线仍带有一定的启发式色彩。
