## 摘要
在长上下文场景下，预填充（Prefilling）的计算开销构成了大语言模型（LLM）和大模型多模态模型（LMM）的主要瓶颈。虽然 Token 剪枝可以缩短序列长度，但现有方法往往依赖于启发式规则，这破坏了与 FlashAttention 等硬件高效算子的兼容性。在这项工作中，我们观察到 Token 会向“语义不动点”（semantic fixing points）演化，使得进一步的处理变得冗余。为此，我们提出了增量注意力选择性停止（Delta Attention Selective Halting，简称 DASH），这是一种无需训练的策略，通过监测自注意力机制的逐层更新动态，选择性地停止已趋于稳定的 Token 的计算。广泛的评估证实，DASH 在语言和视觉基准测试中均具有良好的泛化性，在保持模型准确性和硬件效率的同时，显著提升了预填充速度。代码将发布于 https://github.com/verach3n/DASH.git。

## Abstract
Prefilling computational costs pose a significant bottleneck for Large Language Models (LLMs) and Large Multimodal Models (LMMs) in long-context settings. While token pruning reduces sequence length, prior methods rely on heuristics that break compatibility with hardware-efficient kernels like FlashAttention. In this work, we observe that tokens evolve toward \textit{semantic fixing points}, making further processing redundant. To this end, we introduce Delta Attention Selective Halting (DASH), a training-free policy that monitors the layer-wise update dynamics of the self-attention mechanism to selectively halt stabilized tokens. Extensive evaluation confirms that DASH generalizes across language and vision benchmarks, delivering significant prefill speedups while preserving model accuracy and hardware efficiency. Code will be released at https://github.com/verach3n/DASH.git.

### 1. 核心问题与整体含义（研究动机和背景）
在处理长文本或高分辨率视觉输入时，大语言模型（LLM）和多模态模型（LMM）的**预填充（Prefilling）阶段**面临巨大的计算压力。虽然 Token 剪枝可以减少序列长度，但现有方法存在两个主要痛点：
*   **硬件不兼容性**：大多数剪枝方法依赖于启发式重要性分数（如累积注意力权重），这需要显式计算完整的注意力矩阵，从而破坏了与 FlashAttention 等硬件加速算子的兼容性。
*   **计算冗余**：作者观察到，随着模型层数的加深，许多 Token 的表示会趋于稳定（即进入“语义固定点”），继续对其进行复杂的注意力计算和 FFN 处理带来的边际收益极低。

### 2. 方法论：DASH 核心思想与技术细节
DASH（Delta Attention Selective Halting）的核心思想是：**通过监测 Token 的更新幅度来识别冗余。**

*   **核心指标 ($\Delta_{attn}$)**：在第 $l$ 层，定义 Token $t$ 的更新量为自注意力子层在残差相加前的输出向量 $U^{(l)}_t$ 的 $L_2$ 范数：$\Delta(l)_t = \|U^{(l)}_t\|_2$。
*   **选择性停止策略**：
    1.  **正常处理**：在起始层 $l_s$ 之前，所有 Token 正常通过每一层。
    2.  **单次判定（Single-shot）**：在指定的起始层 $l_s$，计算所有 Token 的 $\Delta_{attn}$ 分数。
    3.  **保留与停止**：根据预设的剪枝比例 $\rho$，保留分数最高的 Top-K 个 Token 构成“活跃集”，其余 Token 被判定为已稳定并“停止”（Halt）。
    4.  **后续跳过**：在 $l_s$ 之后的所有层中，被停止的 Token 不再参与自注意力和 FFN 计算，其状态保持在 $l_s$ 层的输出值不变。
*   **硬件友好性**：由于 DASH 只需要在 $l_s$ 层计算一次向量范数，不需要物化完整的注意力矩阵，因此可以完美兼容 FlashAttention。

*   **语义固定点确实存在**：大多数 Token 在中间层后更新量极小，且这些 Token 在后续层中也很少被其他 Token 关注。
*   **视觉 Token 饱和更快**：在多模态模型中，视觉 Token 比文本 Token 更早达到稳定状态。

### 7. 优点
*   **无需训练（Training-free）**：即插即用，不需要对现有模型进行微调。
*   **硬件高效**：与现代推理加速库（如 FlashAttention）高度兼容，能带来真实的端到端加速，而非仅仅是理论上的 FLOPs 降低。
*   **通用性强**：一套逻辑同时适用于纯文本长上下文和多模态视觉输入。
*   **直觉清晰**：基于“稳定性意味着冗余”的观察，具有很强的解释性。

### 8. 不足与局限
*   **起始层敏感性**：虽然论文提供了一个基于困惑度（PPL）的代理指标来选择起始层 $l_s$，但最佳 $l_s$ 在不同模型和任务间仍可能存在波动。
*   **任务偏差**：在需要极度精细局部信息的任务（如 OCRBench、ChartQA）上，虽然 DASH 依然领先，但领先优势较通用任务有所缩小。
*   **KV Cache 策略**：DASH 主要针对预填充阶段的计算，虽然减少了 KV Cache 的长度，但未深入探讨其与解码阶段复杂 Cache 管理策略（如分页注意力）的深度集成。
