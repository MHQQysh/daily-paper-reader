## 摘要
大语言模型彻底改变了自然语言处理领域，在广泛的任务中取得了前所未有的成功。然而，它们在长上下文场景中的实际应用受到 Key-Value (KV) 缓存巨大内存占用的严重阻碍。虽然通道剪枝已成为一种极具前景的压缩策略，但现有方法孤立地评估通道重要性，从根本上忽略了共同决定模型性能的通道间相互作用。这种疏忽导致了次优的剪枝决策。为解决这一问题，我们提出了 GRACE（Graph-guided Adaptive Channel Elimination，图引导的自适应通道消除），这是一个将 KV 缓存压缩重新定义为基于图的优化问题的新颖框架。GRACE 将通道建模为节点，将它们之间的相互作用建模为加权边，通过最小化注意力权重矩阵的重构误差，从而识别出用于剪枝的近乎最优的通道子集。此外，GRACE 结合了一种自适应保护机制，防止显著的 Key 通道被移除，从而确保了鲁棒的自回归解码过程。广泛的实验表明，GRACE 可以在性能下降微乎其微的情况下将 KV 缓存大小减少 60%，且表现持续优于现有最先进的方法。

## Abstract
Large Language Models have revolutionized natural language processing, achieving unprecedented success across a vast range of tasks. However, their practical application in long-context scenarios is severely hampered by the formidable memory footprint of the Key-Value cache. While channel pruning has emerged as a promising compression strategy, existing methods evaluate channel importance in isolation, fundamentally ignoring the inter-channel interactions that collectively dictate model performance. This oversight leads to suboptimal pruning decisions. To address this, we introduce \textbf{GRACE} (\textbf{GR}aph-guided \textbf{A}daptive \textbf{C}hannel \textbf{E}limination), a novel framework that reframes KV cache compression as a graph-based optimization problem. GRACE models channels as nodes and their interactions as weighted edges, enabling the identification of a near-optimal channel subset for pruning by minimizing the reconstruction error of the attention weight matrix. Furthermore, GRACE incorporates an adaptive protection mechanism that shields salient key channels from removal, ensuring a robust autoregressive decoding process. Extensive experiments show that GRACE can reduce KV cache size by 60\% with negligible performance degradation, consistently outperforming the state-of-the-art method.

### 1. 核心问题与研究动机
*   **背景**：大语言模型（LLMs）在处理长文本时，Key-Value (KV) Cache 的内存占用随序列长度线性增长，成为推理效率的主要瓶颈。
*   **核心问题**：现有的通道剪枝方法（如 THINK）通常**孤立地评估每个通道的重要性**，忽略了通道之间的二阶相互作用（协方差），导致剪枝后的注意力矩阵重构误差较大，模型性能受损。
*   **研究动机**：通过引入图论建模，捕捉通道间的复杂交互，并解决推理阶段 Query 向量分布偏移带来的鲁棒性问题。

### 2. 方法论：GRACE 框架
*   **核心思想**：将 KV Cache 压缩建模为一个图优化问题，通过最小化注意力权重矩阵的 Frobenius 范数重构误差来选择最优通道子集。
*   **关键技术细节**：
    *   **图论建模**：将 $d$ 个通道视为图的节点。节点权重代表通道的“自重要性”，边权重代表通道间的“交互项”。总重构误差被分解为所有被剪枝通道的节点权重与边权重之和。
    *   **MIES 算法（最小增量误差选择）**：由于寻找最小权重诱导子图是 NP-hard 问题，论文提出了一种贪心算法。该算法从全集开始，迭代地移除那些对累积重构误差贡献最小的通道，动态更新剩余通道的得分。
    *   **离群值保护机制（Salient Channel Protection）**：识别并预先屏蔽（不参与剪枝）具有巨大模长的“离群通道”。这些通道通常是模型稳定性的支柱，且能缓冲未来推理步骤中 Query 向量的波动。
    *   **无需训练**：该方法是 Plug-and-Play（即插即用）的，不需要离线微调或训练掩码。

### 6. 主要结论与发现
*   **性能提升**：GRACE 在所有测试任务中一致优于 THINK，尤其在浅层和深层网络中，重构误差降低了 30-40%。
*   **高压缩率**：在减少 60% 通道的情况下，模型性能下降极小。在“大海捞针”测试中，即使在极度受限的 96-token 缓存下，GRACE 的检索准确率也显著高于对比方法。
*   **推理效率**：虽然引入了图计算，但相比 THINK，首字延迟（TTFT）仅略微增加，而生成速度（TPOT）几乎持平，证明了其在实际部署中的可行性。

### 7. 优点与亮点
*   **理论严密**：通过数学分解证明了通道交互项不可忽略，将剪枝问题转化为图论问题具有创新性。
*   **鲁棒性强**：自适应保护机制有效解决了 LLM 中常见的“巨大激活值（Massive Activations）”问题，增强了对推理时分布偏移的抵抗力。
*   **通用性**：作为一种通道维度的压缩手段，它可以与现有的 Token 级压缩（如 SnapKV）或量化技术正交结合，进一步提升压缩极限。

### 8. 不足与局限
*   **计算复杂度**：虽然 TPOT 影响较小，但 MIES 算法在处理极高维度通道时，其 $O(n_{prune} \cdot d)$ 的贪心选择过程可能在超大规模模型（如 70B+）上产生更明显的前处理延迟。
*   **实验覆盖**：实验主要集中在 8B 及以下规模的模型，尚未验证在超大规模参数模型（如 Llama-3-405B）上的表现。
*   **应用限制**：该方法主要针对 Prefilling 阶段产生的初始 KV Cache 进行压缩，对于极长序列生成过程中动态增长的 KV 增量，其保护机制的长期有效性仍有待进一步探讨。
