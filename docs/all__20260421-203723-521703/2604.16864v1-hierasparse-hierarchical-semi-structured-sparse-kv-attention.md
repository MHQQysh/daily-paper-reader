## 摘要
长上下文大语言模型（LLMs）的部署面临着巨大挑战，这主要源于自注意力机制的高昂计算成本以及键值缓存（KV Cache）巨大的内存开销。在本文中，我们提出了 HieraSparse，这是一个分层 KV Cache 压缩框架，配备了利用 GPU 稀疏张量核心（sparse tensor cores）的加速算子，旨在加速预填充（prefill）和解码（decode）阶段的半结构化 KV Cache 注意力计算。通过分层设计，我们的方法实现了灵活的质量-稀疏度权衡，并成功地将稀疏性转化为效率。与利用非结构化稀疏性的最先进解码方法相比，在相同稀疏度水平下，HieraSparse 实现了 1.2 倍的 KV 压缩率和 4.57 倍的注意力加速。此外，我们将半结构化 KV Cache 剪枝扩展到了预填充阶段，在最高稀疏度下展示了高达 1.85 倍的注意力加速。最后，我们使用一种简单的基于幅值的剪枝方法评估了 HieraSparse 的生成质量，结果表明在没有显著质量下降的情况下，可以实现 1.37 倍的预填充加速和 1.77 倍的解码加速。代码库可在 https://github.com/psl-ntu/HieraSparse 找到。

## Abstract
The deployment of long-context Large Language Models (LLMs) poses significant challenges due to the intense computational cost of self-attention and the substantial memory overhead of the Key-Value Cache (KV Cache). In this paper, we introduce HieraSparse, a hierarchical KV Cache compression framework with acceleration kernels that leverage GPU sparse tensor cores to speed up semi-structured KV Cache attention for both the prefill and decode phases. With the hierarchical design, our method allows for a flexible quality-sparsity trade-off and successfully converts sparsity into efficiency. Compared to the state-of-the-art decode method that utilizes unstructured sparsity, HieraSparse achieves $\mathbf{1.2\times}$ KV compression ratio and $\mathbf{4.57\times}$ attention speedup at the same sparsity level. Furthermore, we extended the semi-structured KV Cache pruning to the prefill stage, which demonstrated up to $\mathbf{1.85\times}$ attention speedup at the highest sparsity. Lastly, we evaluate the generation quality of HieraSparse with a simple magnitude-based pruning method, and the results show that $\mathbf{1.37\times}$ prefill speedup and $\mathbf{1.77\times}$ decode speedup can be achieved without significant quality drop. The codebase can be found at https://github.com/psl-ntu/HieraSparse.

随着大语言模型（LLM）支持的上下文窗口不断延长（从数十万到上百万 token），推理过程面临两大瓶颈：
*   **计算瓶颈**：自注意力机制的复杂度随序列长度呈平方级增长，在长文本预填充阶段，注意力计算可占据总延迟的 80% 以上。
*   **内存瓶颈**：KV Cache 的内存占用随长度线性增长，长文本场景下其内存需求往往超过单张 GPU 的容量。
现有的 KV Cache 压缩方法要么是**粗粒度**的（如丢弃整个 token），灵活性差；要么是**细粒度非结构化**的，虽然稀疏度高，但由于硬件不支持非结构化稀疏加速，无法转化为实际的推理提速。本文旨在利用现代 GPU 的 **N:M 半结构化稀疏张量核心（Sparse Tensor Cores）**，实现既能压缩内存又能显著加速计算的 KV 注意力框架。

### 2. 方法论：HieraSparse 框架
HieraSparse 的核心思想是结合**分层块管理**与**硬件原生的半结构化稀疏加速**。

*   **分层缓存剪枝（Hierarchical Pruner）**：
    *   将 KV Cache 划分为块（Blocks）。
    *   **块级掩码**：识别重要区域（如 Attention Sinks、局部窗口、Heavy Hitters），将其设为稠密块；其余设为稀疏块。
    *   **元素级掩码**：在稀疏块内部，应用 N:M 约束（如 2:4 稀疏），即每 M 个连续元素中只保留 N 个。
*   **高效压缩算子（Cache Compressor）**：
    *   设计了近乎零开销的在线压缩内核，将原始稠密张量转换为非零值数据和元数据（Metadata），并存储在专门的内存池中。
*   **GPU 加速内核设计**：
    *   **双重转置方案（Trans-Both）**：为了让稀疏张量核心生效（它要求第一个乘数必须是稀疏的），作者重新设计了注意力计算流，通过转置 Key 和 Value，使得预填充和解码阶段都能利用硬件加速。
    *   **寄存器级重排（In-fragment Re-layout）**：利用 `movmatrix` 指令在寄存器内完成数据重排，避免了昂贵的共享内存交换。
    *   **异步流水线**：使用 `cp.async` 指令隐藏内存读取延迟。

### 6. 主要结论与发现
*   **效率提升**：在相同稀疏度下，HieraSparse 比非结构化方法（MUSTAFAR）快 **4.57 倍**，并实现了 **1.2 倍** 的额外压缩率。
*   **预填充加速**：首次将半结构化稀疏成功应用于预填充阶段，最高实现 **1.85 倍** 的算子加速。

*   **敏感度差异**：实验发现 Key Cache 对剪枝比 Value Cache 更敏感，建议在预填充阶段保持 Key 稠密以维持精度。

### 7. 优点
*   **硬件原生加速**：不同于以往“只压缩不加速”的稀疏研究，HieraSparse 真正利用了 Tensor Core 的硬件特性。
*   **全阶段覆盖**：同时优化了预填充（TTFT）和解码（TPOT）两个阶段，这在 KV 压缩研究中较为少见。
*   **极低开销**：在线压缩内核设计精巧，压缩耗时仅占预填充总耗时的 0.5%，几乎可以忽略。

### 8. 不足与局限
*   **剪枝算法简单**：目前默认使用简单的基于幅值（Magnitude-based）的剪枝，可能不是最优的精度保持方案。
*   **硬件依赖**：高度依赖支持 N:M 稀疏指令的现代 GPU（如 Ampere、Hopper 架构），在旧硬件或非 NVIDIA 平台上无法发挥优势。
*   **模型敏感度**：实验显示 Mistral 模型对结构化剪枝比 Llama 更敏感，说明该方法在不同架构模型上的泛化性仍有微小差异，可能需要针对性调优。
