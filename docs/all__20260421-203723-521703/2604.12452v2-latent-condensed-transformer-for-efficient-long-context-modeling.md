## 摘要
大语言模型（LLMs）在处理长上下文时面临重大挑战，主要源于键值（KV）缓存的线性增长和自注意力机制的平方复杂度。现有方法分别解决这些瓶颈：多头潜空间注意力（MLA）通过将 token 投影到低维潜空间来减少 KV 缓存，而稀疏注意力则减少了计算量。然而，稀疏方法无法原生运行在 MLA 的压缩潜空间结构上，从而错失了联合优化的机会。在本文中，我们提出了潜空间压缩注意力（LCA），它直接在 MLA 的潜空间内压缩上下文，其中表示被解耦为语义潜向量和位置键。LCA 通过查询感知池化分别聚合语义向量，并通过锚点选择保留位置键。该方法在不增加参数的情况下，同时降低了计算成本和 KV 缓存。除了 MLA，LCA 的设计与架构无关，可以轻松扩展到 GQA 等其他注意力机制。在理论上，我们证明了一个与长度无关的误差界限。实验表明，LCA 在 128K 上下文下实现了高达 2.5 倍的预填充加速和 90% 的 KV 缓存减少，同时保持了具有竞争力的性能。

## Abstract
Large language models (LLMs) face significant challenges in processing long contexts due to the linear growth of the key-value (KV) cache and quadratic complexity of self-attention. Existing approaches address these bottlenecks separately: Multi-head Latent Attention (MLA) reduces the KV cache by projecting tokens into a low-dimensional latent space, while sparse attention reduces computation. However, sparse methods cannot operate natively on MLA's compressed latent structure, missing opportunities for joint optimization. In this paper, we propose Latent-Condensed Attention (LCA), which directly condenses context within MLA's latent space, where the representation is disentangled into semantic latent vectors and positional keys. LCA separately aggregates semantic vectors via query-aware pooling and preserves positional keys via anchor selection. This approach jointly reduces both computational cost and KV cache without adding parameters. Beyond MLA, LCA's design is architecture-agnostic and readily extends to other attention mechanisms such as GQA. Theoretically, we prove a length-independent error bound. Experiments show LCA achieves up to 2.5$\times$ prefilling speedup and 90% KV cache reduction at 128K context while maintaining competitive performance.

### 1. 核心问题与整体含义（研究动机和背景）
随着 LLM 应用于长文档理解和多轮对话，模型面临两大挑战：
*   **KV 缓存（KV Cache）的线性增长**：导致显存占用随文本长度增加而剧增。
*   **自注意力的平方计算复杂度**：导致长文本的预填充（Prefilling）和推理速度变慢。

目前，DeepSeek 提出的 **MLA（多头潜空间注意力）** 通过低秩投影减少了 KV 缓存，但仍需对所有 token 进行全量计算。而现有的**稀疏注意力**方法通常在还原后的全维空间操作，无法直接利用 MLA 的压缩特性。论文的动机在于：**如何在 MLA 的潜空间内直接进行上下文压缩，从而同时降低计算量和存储开销？**

### 2. 方法论：核心思想与技术细节
LCA 的核心思想是**在潜空间内对上下文进行结构化压缩**，其关键在于处理 MLA 中“解耦”的语义信息和位置信息：

*   **双路径分组压缩策略**：将历史上下文划分为若干个组（Group），每组大小为 $g$。
*   **语义压缩（加权池化）**：针对语义潜向量 $C_{KV}$，利用“查询感知（Query-aware）”的权重进行池化。通过计算当前查询与组内 token 的相关性得分，生成一个代表性的语义向量。理论证明，这种加权池化在最小化重构误差方面是优化的。
*   **位置锚点选择（Max-selection）**：针对带有 RoPE（旋转位置编码）的位置键 $K_R$，由于位置编码具有高度非线性，直接池化会破坏几何关系。LCA 选择组内得分最高的 token 作为“锚点”，保留其原始位置信息。
*   **局部窗口保留**：为了保证近期上下文的精确性，LCA 对最近的 $w$ 个 token 不做压缩，保持全精度。
*   **算法流程**：在预填充阶段，将压缩后的代表性 KV 与局部全精度 KV 融合进行计算；在解码阶段，在线更新缓存，每当新生成的 token 达到组大小 $g$ 时，即触发一次压缩。

### 6. 主要结论与发现
*   **显著加速与减存**：在 128K 上下文下，LCA 实现了 **2.5 倍的预填充加速**和 **90% 的 KV 缓存减少**。

*   **架构无关性**：LCA 的设计可以无缝迁移到 GQA 架构，在 Qwen-7B 上同样实现了 3.25 倍加速和 93% 的缓存节省。

### 7. 优点（亮点）
*   **原生潜空间操作**：首次提出直接在 MLA 潜空间进行压缩，避免了全维重构的开销。
*   **语义与位置解耦处理**：针对语义和位置信息的不同特性采用不同的压缩策略（池化 vs 锚点），设计精巧且符合直觉。
*   **无需额外参数**：通过轻量化微调即可集成到现有预训练模型中，部署成本低。
*   **理论保障**：证明了误差界限与上下文长度无关，为长文本建模提供了理论支撑。

### 8. 不足与局限
*   **内核依赖**：为了达到论文所述的极致加速，必须依赖自定义的 Triton 内核，这增加了在不同推理框架（如 vLLM, TensorRT-LLM）中的迁移难度。
*   **特定任务的微小退化**：在需要极高精度 token 检索的任务（如多文档 QA）中，激进的压缩会导致轻微的性能下降。
*   **量化兼容性未测**：目前实验主要基于 bfloat16，尚未深入探讨该方法与 INT8/FP8 等低精度量化技术的协同效果。
