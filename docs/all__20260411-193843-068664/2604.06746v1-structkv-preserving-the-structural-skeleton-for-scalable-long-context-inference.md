## 摘要
随着大语言模型（LLMs）扩展到支持超过一百万 token 的上下文窗口，键值（KV）缓存的线性增长带来了严重的显存容量和带宽瓶颈，限制了长上下文推理的效率。现有的压缩方法通常根据局部显著性指标对 token 进行优先级排序，以实现预填充计算与解码内存的解耦。然而，这些方法往往依赖于特定层的局部显著性快照，从而系统性地丢弃了那些在整个网络深度中充当全局信息枢纽、但在选定的剪枝层中暂时处于休眠状态的 token。为了解决这一局限性，我们提出了 StructKV，这是一个结构感知的 KV 缓存压缩框架，它引入了三项核心创新：首先，全局入度中心性（Global In-Degree Centrality）通过聚合整个网络深度的注意力模式来识别全局信息枢纽；其次，动态枢轴检测（Dynamic Pivot Detection）利用信息论指标自适应地定位压缩的最佳层；最后，结构化传播与解耦（Structural Propagation and Decoupling）将计算预算与内存存储预算分离。在 LongBench 和 RULER 基准测试上的实验结果表明，StructKV 有效地保留了长程依赖关系和检索鲁棒性。

## Abstract
As Large Language Models (LLMs) scale to support context windows exceeding one million tokens, the linear growth of Key-Value (KV) cache imposes severe memory capacity and bandwidth bottlenecks, constraining the efficiency of long-context inference. Existing compression approaches typically prioritize tokens based on local saliency metrics to decouple prefill computation from decoding memory. However, these methods often rely on local saliency snapshots at a specific layer, thereby systematically discarding tokens that act as global information hubs across the network depth but appear temporarily dormant at the specific layer selected for pruning. To address this limitation, we propose StructKV, a structure-aware KV cache compression framework that introduces three core innovations: First, Global In-Degree Centrality aggregates attention patterns across the network depth to identify global information hubs. Second, Dynamic Pivot Detection utilizes information-theoretic metrics to adaptively locate the optimal layer for compression. Finally, Structural Propagation and Decoupling separates the computational budget from the memory storage budget. Experimental results on the LongBench and RULER benchmarks demonstrate that StructKV effectively preserves long-range dependencies and retrieval robustness.

随着大语言模型（LLM）支持的上下文窗口突破百万级，**KV Cache（键值缓存）** 的线性增长成为了推理效率的致命瓶颈。这主要体现在两个阶段：
*   **预填充阶段（Prefill）：** 注意力机制的平方级计算复杂度导致高延迟。
*   **解码阶段（Decoding）：** 庞大的 KV Cache 消耗极高显存并限制了系统吞吐量。
**现有方法的局限：** 现有的压缩技术（如 SnapKV, FastKV）通常依赖于特定层的“局部显著性快照”来决定保留哪些 token。作者发现这种做法是“近视”的，容易误删那些在全局结构中至关重要、但在特定层表现不活跃的“信息枢纽” token，从而导致长文本理解能力的下降。

### 2. 方法论：核心思想与关键技术
StructKV 提出了一个**结构感知**的压缩框架，包含三大核心创新：
*   **全局入度中心性（Global In-Degree Centrality）：** 不再只看单层注意力分数，而是通过递归累加从输入层到“枢轴层”的所有注意力模式。这种方法将 token 视为图节点，计算其在整个网络深度中的累积贡献，从而识别出支撑语义流的“结构骨架”。
*   **动态枢轴检测（Dynamic Pivot Detection）：** 放弃了固定在某一层进行剪枝的传统做法。StructKV 引入了一个在线检测机制，通过监控注意力熵（Entropy）、稀疏度（Sparsity）和方差（Variance）的梯度变化，自动捕捉模型从“广泛探索”转向“聚焦提取”的相位转换点（Pivot Layer）。
*   **结构化传播与解耦（Structural Propagation & Decoupling）：** 
    *   **计算解耦：** 在枢轴层之后，仅传播保留的结构化 token，降低深层的计算负载。
    *   **存储解耦：** 独立设置计算预算（$R_{struct}$）和存储预算（$R_{KV}$），允许在保持较高计算精度的同时，极度压缩用于解码的显存占用。

### 6. 主要结论与发现
*   **性能卓越：** 在仅保留 10% KV 缓存的情况下，StructKV 在 LongBench 上的表现接近全上下文模型，且显著优于 FastKV 和 GemFilter。
*   **深度自适应：** 实验发现模型越大，注意力稳定的“相位转换点”越靠后（如 Qwen-32B 在第 28 层才稳定），StructKV 的动态检测成功捕捉到了这一规律。
*   **检索鲁棒性：** 在 128K 长度的 RULER 测试中，StructKV 几乎消除了 FastKV 出现的性能骤降，证明了全局中心性指标对关键信息的保护作用。

### 7. 优点（亮点）
*   **全局视角：** 突破了局部显著性的局限，通过累积注意力分数找准了文本的“语义骨架”。
*   **高度自适应：** 动态枢轴检测解决了不同架构模型需要手动调参剪枝层的问题，具有很强的通用性。
*   **高效解耦：** 巧妙地平衡了预填充速度和解码显存，实现了推理全流程的优化。

### 8. 不足与局限
*   **验证长度上限：** 受限于计算资源，实验最高验证到 128K token，尚未在百万级（1M+）真实场景下进行超大规模验证。
*   **架构覆盖：** 目前主要针对标准的稠密 Transformer 架构，对于混合专家模型（MoE）或非注意力架构（如 SSM/Mamba）的适用性有待进一步研究。
*   **硬件依赖：** 虽然计算开销低，但特定的向量聚合操作在显存带宽受限的低端硬件上可能存在优化空间。
