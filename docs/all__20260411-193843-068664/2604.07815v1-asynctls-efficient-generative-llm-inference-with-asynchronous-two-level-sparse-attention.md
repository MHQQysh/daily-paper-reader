## 摘要
大语言模型（LLM）的长文本推理面临着注意力机制的平方级复杂度和极高的 KV 缓存（KV cache）内存占用这两大挑战。虽然 Token 级稀疏注意力提供了卓越的准确度，但其索引开销巨大；块级方法虽能提高效率，却以牺牲精度为代价。我们提出了 AsyncTLS，这是一种分层稀疏注意力系统，它结合了粗粒度的块过滤与细粒度的 Token 选择，以平衡准确度与效率，并辅以一个异步卸载引擎，通过利用时间局部性实现 KV 缓存传输与计算的重叠。在 Qwen3 和 GLM-4.7-Flash 上针对 GQA 和 MLA 架构进行的评估表明，AsyncTLS 在 48k - 96k 上下文长度下，实现了与全量注意力相当的准确度，同时带来了 1.2 倍至 10.0 倍的算子加速以及 1.3 倍至 4.7 倍的端到端吞吐量提升。

## Abstract
Long-context inference in LLMs faces the dual challenges of quadratic attention complexity and prohibitive KV cache memory. While token-level sparse attention offers superior accuracy, its indexing overhead is costly; block-level methods improve efficiency but sacrifice precision. We propose AsyncTLS, a hierarchical sparse attention system that combines coarse-grained block filtering with fine-grained token selection to balance accuracy and efficiency, coupled with an asynchronous offloading engine that overlaps KV cache transfers with computation via temporal locality exploitation. Evaluated on Qwen3 and GLM-4.7-Flash across GQA, and MLA architectures, AsyncTLS achieves accuracy comparable to full attention while delivering 1.2x - 10.0x operator speedups and 1.3x - 4.7x end-to-end throughput improvements on 48k - 96k contexts.

*   **研究动机**：
    *   **Token 级稀疏性**：精度高，但每一步推理都要计算全局重要性，索引开销（Indexing Overhead）极大。
    *   **块级（Block-level）稀疏性**：效率高，但粗粒度的过滤会引入噪声或丢失关键信息，导致模型精度下降。
    *   **KV 卸载（Offloading）**：将缓存移至 CPU 内存可解决容量问题，但 PCIe 带宽限制了数据传输速度，导致严重的推理延迟。

### 2. 方法论：AsyncTLS 核心思想与技术细节
AsyncTLS 提出了**层次化稀疏注意力**与**异步卸载引擎**相结合的方案：
*   **两级稀疏注意力机制**：
    1.  **粗粒度块过滤（Level-1）**：将 KV 缓存划分为块，利用优化后的 GEMM 算子快速计算块重要性得分，筛选出潜在相关的 Top-K 块。

*   **异步卸载引擎**：
    1.  **时间局部性预测**：利用注意力模式在相邻时间步之间的稳定性，使用第 $t-1$ 步的块过滤结果来指导第 $t$ 步的 Token 选择。
    2.  **异步预取（Prefetching）**：在计算当前步的注意力时，异步从 CPU 预取下一步所需的 KV 块，实现计算与数据传输的重叠（Overlap）。
    3.  **增量传输**：仅传输在连续步骤中发生变化的 KV 块，最大限度减少 PCIe 带宽占用。
