## 摘要
大语言模型 (LLMs) 在处理长提示词时会产生显著的计算和内存开销，因为全自注意力机制的复杂度随输入长度呈二次方增长。Token 压缩旨在通过减少表示输入的 token 数量来应对这一挑战。然而，现有的提示词压缩方法主要在 token 空间中运行，忽略了潜嵌入空间中的低效问题。在本文中，我们提出了 K-Token 合并（K-Token Merging），这是一种潜空间压缩框架，通过轻量级编码器将每个连续的 K 个 token 嵌入块合并为单个嵌入。压缩后的序列由经过 LoRA 适配的大语言模型处理，而生成过程仍保持在原始词表空间。在结构化推理 (Textualized Tree)、情感分类 (Amazon Reviews) 和代码编辑 (CommitPackFT) 上的实验表明，K-Token 合并在性能与压缩比的权衡中处于帕累托前沿，在性能下降极小的情况下实现了高达 75% 的输入长度缩减。

## Abstract
Large Language Models (LLMs) incur significant computational and memory costs when processing long prompts, as full self-attention scales quadratically with input length. Token compression aims to address this challenge by reducing the number of tokens representing inputs. However, existing prompt-compression approaches primarily operate in token space and overlook inefficiencies in the latent embedding space. In this paper, we propose K-Token Merging, a latent-space compression framework that merges each contiguous block of K token embeddings into a single embedding via a lightweight encoder. The compressed sequence is processed by a LoRA-adapted LLM, while generation remains in the original vocabulary. Experiments on structural reasoning (Textualized Tree), sentiment classification (Amazon Reviews), and code editing (CommitPackFT) show that K-Token Merging lies on the Pareto frontier of performance vs. compression, achieving up to 75% input length reduction with minimal performance degradation.

### 2. 方法论：K-Token Merging
*   **核心思想**：利用一个轻量级的编码器，将输入序列中每 $K$ 个连续 Token 的嵌入向量合并为一个单一的压缩嵌入向量，从而在输入端直接将序列长度缩减为原来的 $1/K$。
*   **关键技术细节**：
    *   **Prefill（预填充）阶段**：编码器 $f$（一个约 50MB 的小型 MLP）接收 $K$ 个原始嵌入，输出一个压缩嵌入 $C_i$。
    *   **Generation（生成）阶段**：模型在压缩后的前缀基础上，以自回归方式生成原始词表中的 Token。生成的 Token 不再压缩，直接追加到序列中。
    *   **LoRA 适配**：使用 LoRA 适配器对 LLM 进行微调，使其能够理解并处理这些非标准的压缩嵌入。
    *   **平均值初始化（Average-Initialized Encoder）**：为了加速收敛，编码器被设计为输出“$K$ 个 Token 嵌入的平均值”与“MLP 残差”之和。初始状态下 MLP 权重接近零，使模型从语义合理的平均值开始学习。

### 7. 优点与亮点
*   **潜空间视角**：不同于传统的 Token 级操作，该方法切中了嵌入空间表示效率低下的痛点。
*   **通用性强**：不需要像“词表扩展”那样处理长尾 Token 爆炸问题，能够自然泛化到未见的 $K$-gram 组合。
*   **初始化巧妙**：通过残差结构和平均值初始化，解决了潜空间压缩训练初期不稳定的问题。
*   **端到端兼容**：可以与现有的硬压缩方法结合使用，具有互补性。

### 8. 不足与局限
*   **生成阶段未压缩**：目前仅压缩输入（Prefill），生成过程中的 KV Cache 仍会随输出长度增长，限制了长文本生成的效率提升。

*   **模型规模限制**：实验仅在 0.5B 模型上完成，缺乏在 7B、70B 等主流大模型上的验证。
*   **微调成本**：该方法需要针对特定任务或分布进行 LoRA 微调，不像某些硬压缩方法那样可以“即插即用”。
