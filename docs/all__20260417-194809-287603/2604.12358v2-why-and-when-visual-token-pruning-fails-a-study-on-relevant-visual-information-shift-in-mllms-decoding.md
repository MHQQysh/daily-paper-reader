## 摘要
最近，视觉 Token 剪枝已被用于处理多模态大语言模型（MLLMs）中海量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但在有效泛化到复杂的视觉推理任务时却面临困难，这是以往研究中尚未充分探讨的关键空白。通过系统性分析，我们发现解码过程中的相关视觉信息偏移（RVIS）是导致失效的主要诱因。为解决这一问题，我们提出了解码阶段偏移感知 Token 剪枝（DSTP），这是一个无需训练的插件式框架，使现有的剪枝方法能够在解码阶段将视觉 Token 与不断变化的推理需求相对齐。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能退化，同时在视觉理解基准测试中也持续带来了性能提升。此外，DSTP 在多种先进架构上均证明了其有效性，突显了其泛化能力以及极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

#### 1. 核心问题与整体含义（研究动机和背景）
随着多模态大语言模型（MLLM）的发展，视觉 Token 数量的激增带来了巨大的计算和内存开销。现有的视觉 Token 剪枝方法（如 FastV, DivPrune 等）虽然在简单的视觉问答（VQA）任务中表现良好，但在复杂的**视觉推理（VMR）**任务（如数学竞赛、逻辑谜题）中性能会剧烈下降。
论文通过诊断分析发现，这种失效的根源在于**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**：在复杂推理过程中，模型在不同解码步骤中关注的视觉区域会发生动态变化，而现有的静态剪枝方法在预填充（Prefill）阶段就永久丢弃了后续步骤可能需要的关键 Token。

#### 2. 方法论：DSTP 框架
作者提出了**解码阶段偏移感知 Token 剪枝（DSTP）**，这是一个无需训练、即插即用的插件框架。其核心流程如下：
*   **RVIS 检测模块 (RISD)：** 在解码过程中，实时监控当前生成 Token 对视觉 Token 的注意力分布。通过计算当前注意力向量与预填充阶段锚点注意力向量的余弦相似度，当相似度低于设定阈值 $\tau$ 时，判定发生了 RVIS。
*   **上下文保留视觉 Token 交换 (CPTS)：** 一旦检测到偏移，模型会重新评估所有原始视觉 Token（包括之前被剪枝掉的）。
    *   **动态重选：** 根据当前查询（Query）选出新的 Top-K 个相关 Token。
    *   **上下文保留：** 为了防止生成中断或逻辑不连贯，不直接替换，而是将新选出的 Token 与原始剪枝 Token 取**并集**，形成一个临时的扩展 Token 集。
    *   **持续时间控制：** 该扩展集仅维持 $L$ 个解码步骤，随后恢复到原始剪枝状态，以平衡精度与效率。

*   **硬件：** 使用了 **NVIDIA L40S GPU** 进行实验。
*   **算力说明：** 论文明确指出该方法是**免训练（Training-free）**的。在推理效率方面，作者提供了详细的 TFLOPs 计算和延迟（Latency）分析。虽然 DSTP 在检测到偏移时会短暂增加 Token 数量，但由于其减少了模型因信息缺失而产生的冗余推理（生成更短、更准确的回答），在某些配置下，其端到端延迟甚至低于原始剪枝方法。

*   **实验规模：** 涵盖了 2 种主流架构、3 种代表性剪枝基线、8 个多模态基准测试。
*   **消融实验：** 针对 RVIS 检测阈值 $\tau$、上下文保留长度 $L$、Token 交换策略（硬替换 vs. 并集）以及不同 RVIS 频率下的成功率进行了深入分析。
*   **客观性：** 实验不仅关注准确率，还对比了 TFLOPs、每秒生成 Token 数（TPS）和显存占用，评估非常全面且客观。

#### 6. 主要结论与发现
*   **RVIS 是失效主因：** 复杂推理任务中 RVIS 发生频率极高，而 VQA 任务中注意力相对固定。
*   **动态恢复的必要性：** 静态剪枝无法应对多步推理中的视觉焦点转移。
*   **DSTP 的有效性：** 在 33.3% 的 Token 保留率下，DSTP 将剪枝模型的推理性能从约 50% 提升至 80%-90% 以上，且计算开销增加极小。
*   **效率增益：** 提供正确的视觉信息能缩短模型的推理路径，从而减少生成的 Token 总数，优化整体推理时间。

#### 7. 优点
*   **洞察深刻：** 首次识别并量化了解码阶段的视觉信息偏移（RVIS）现象。
*   **即插即用：** 无需重新训练模型，能够直接提升现有各种剪枝方法的鲁棒性。
*   **平衡性好：** 通过临时的 Token 并集策略，既保留了全局上下文，又引入了局部细节。

#### 8. 不足与局限
*   **超参数依赖：** 阈值 $\tau$ 和持续时间 $L$ 目前是静态设置的，对于不同难度或类型的题目，可能需要动态调整以达到最优平衡。
*   **层级一致性：** 目前将选定的 Token 统一应用于所有 Transformer 层，未考虑不同层对视觉细节需求的差异（早期层可能需要更多细节，后期层需要语义）。
*   **极端压缩限制：** 在极高的剪枝率下，虽然 DSTP 有提升，但仍难以完全弥补原始信息的丢失。
