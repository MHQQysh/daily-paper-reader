## 摘要
长上下文大语言模型（LLMs）的部署面临着巨大挑战，这主要源于自注意力机制的高昂计算成本以及键值缓存（KV Cache）的庞大内存开销。在本文中，我们介绍了 HieraSparse，这是一个分层 KV Cache 压缩框架，它配备了利用 GPU 稀疏张量核心（sparse tensor cores）的加速算子，旨在加速预填充（prefill）和解码（decode）阶段的半结构化 KV Cache 注意力。通过分层设计，我们的方法实现了灵活的质量-稀疏度权衡，并成功将稀疏性转化为效率。与利用非结构化稀疏性的最先进解码方法相比，在相同稀疏度水平下，HieraSparse 实现了 1.2 倍的 KV 压缩率和 4.57 倍的注意力加速。此外，我们将半结构化 KV Cache 剪枝扩展到了预填充阶段，在最高稀疏度下展示了高达 1.85 倍的注意力加速。最后，我们使用一种简单的基于幅度的剪枝方法评估了 HieraSparse 的生成质量，结果表明在没有显著质量下降的情况下，可以实现 1.37 倍的预填充加速和 1.77 倍的解码加速。代码库可在 https://github.com/psl-ntu/HieraSparse 找到。

## Abstract
The deployment of long-context Large Language Models (LLMs) poses significant challenges due to the intense computational cost of self-attention and the substantial memory overhead of the Key-Value Cache (KV Cache). In this paper, we introduce HieraSparse, a hierarchical KV Cache compression framework with acceleration kernels that leverage GPU sparse tensor cores to speed up semi-structured KV Cache attention for both the prefill and decode phases. With the hierarchical design, our method allows for a flexible quality-sparsity trade-off and successfully converts sparsity into efficiency. Compared to the state-of-the-art decode method that utilizes unstructured sparsity, HieraSparse achieves $\mathbf{1.2\times}$ KV compression ratio and $\mathbf{4.57\times}$ attention speedup at the same sparsity level. Furthermore, we extended the semi-structured KV Cache pruning to the prefill stage, which demonstrated up to $\mathbf{1.85\times}$ attention speedup at the highest sparsity. Lastly, we evaluate the generation quality of HieraSparse with a simple magnitude-based pruning method, and the results show that $\mathbf{1.37\times}$ prefill speedup and $\mathbf{1.77\times}$ decode speedup can be achieved without significant quality drop. The codebase can be found at https://github.com/psl-ntu/HieraSparse.

随着大语言模型（LLM）支持的上下文窗口不断延长（从 128K 到 1M+ 标记），推理过程面临两大瓶颈：
*   **计算开销（Computation）：** 自注意力机制具有 $O(n^2)$ 的复杂度。在长文本预填充阶段，注意力计算可占总延迟的 80% 以上；在解码阶段，随着 KV Cache 增长，注意力计算也逐渐主导耗时。
*   **内存开销（Memory）：** KV Cache 随序列长度线性增长。例如，Llama-3.1-8B 在 1M 上下文下需要约 125 GiB 显存，远超单张现代 GPU 的容量。

### 2. 方法论：核心思想与关键技术
HieraSparse 提出了一个**分层半结构化稀疏 KV 缓存压缩框架**，核心思想是利用现代 GPU（如 NVIDIA Ampere/Hopper）的 **N:M 稀疏张量核心（Sparse Tensor Core）** 来实现真正的硬件加速。

*   **分层剪枝策略（Hierarchical Pruning）：**
    *   **块级（Block-level）：** 将 KV Cache 划分为块，识别并保留关键区域（如 Attention Sinks、Heavy Hitters、Local Windows）为稠密块，其余块标记为稀疏块。
    *   **元素级（Element-level）：** 在稀疏块内部应用 N:M 结构化稀疏（如 2:4 稀疏），即每 M 个连续元素中只保留 N 个非零值。
*   **高效压缩算子（Cache Compressor）：** 设计了近乎零开销的在线压缩算子，将原始张量转换为非零值矩阵和元数据（Metadata），并使用块索引映射（Block Index Map）管理混合的稠密/稀疏内存池。
*   **GPU 加速算子设计：**
    *   **双重转置方案（Trans-Both）：** 为了让 K 和 V 都能利用稀疏张量核心（该核心要求第一个操作数为稀疏），算法将注意力计算重构为计算 $S^T$ 和 $O^T$，从而使 K 和 V 成为稀疏操作数。
    *   **寄存器重排（In-fragment Re-layout）：** 使用 `movmatrix` 指令在寄存器层面完成矩阵转置，避免了昂贵的共享内存交换。
    *   **异步流水线：** 利用 `cp.async` 指令隐藏内存加载延迟。

### 6. 主要结论与发现

### 7. 优点
*   **硬件原生加速：** 突破了以往稀疏注意力“只稀疏不加速”的困境，真正利用了 Tensor Core 的硬件特性。
*   **全阶段覆盖：** 同时优化了预填充（计算密集型）和解码（内存密集型）两个阶段。
*   **灵活性：** 支持混合稠密与稀疏块，允许针对不同层、不同阶段设置不同的稀疏策略。
*   **低开销：** 在线压缩算子效率极高，不会抵消计算带来的收益。

*   **剪枝算法简单：** 目前主要采用基于幅度的简单剪枝，可能未完全挖掘出预填充阶段的加速潜力（如更复杂的离线剪枝可能效果更好）。

*   **模型敏感度：** 不同模型对结构化稀疏的耐受度不同（如 Mistral 表现出更高的敏感性），需要针对特定模型微调稀疏参数。
