## 摘要
最近，视觉 Token 剪枝已被用于处理多模态大语言模型（MLLMs）中海量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但它们难以有效地泛化到复杂的视觉推理任务，这是以往研究中尚未充分探索的关键差距。通过系统分析，我们确定解码过程中的相关视觉信息偏移（RVIS）是导致失效的主要驱动因素。为了解决这一问题，我们提出了解码阶段偏移感知 Token 剪枝（DSTP），这是一个无需训练的附加框架，使现有的剪枝方法能够在解码阶段将视觉 Token 与不断变化的推理需求相对齐。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能下降，同时在视觉理解基准测试中也持续带来性能提升。此外，DSTP 在多种最先进的架构中均表现出有效性，突显了其泛化能力和极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

*   **核心发现**：论文首次指出，复杂推理是一个多步骤过程，模型在解码的不同阶段需要关注图像的不同区域。作者将其定义为**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**。现有的剪枝方法通常在“预填充（Prefill）”阶段一次性剔除 Token，这种静态策略无法应对解码过程中动态变化的视觉需求，导致推理失败。

### 2. 论文提出的方法论：DSTP 框架
作者提出了**解码阶段偏移感知 Token 剪枝（DSTP）**，这是一种无需训练的插件式框架，核心思想是“检测偏移并动态交换”。
*   **RVIS 检测模块 (RISD)**：在解码的每一步，计算当前文本查询对视觉 Token 的注意力分布，并将其与预填充阶段的初始注意力分布进行余弦相似度比较。当相似度低于设定阈值 $\tau$ 时，触发 RVIS 预警。
*   **上下文保留视觉 Token 交换 (CPTS)**：
    *   **重新评估**：一旦检测到偏移，从之前被剔除的“候选池”中重新找回当前最相关的 $k$ 个 Token。
    *   **合并与保留**：为了保持生成的连贯性，不直接替换，而是将初始 Token 集与新找回的 Token 集取并集，形成一个临时的扩展 Token 集。
    *   **持续时间控制**：该扩展集会维持 $L$ 个解码步，随后恢复到初始剪枝状态，以平衡性能与效率。

### 6. 主要结论与发现
*   **RVIS 是失效主因**：推理任务中 RVIS 发生频率远高于理解任务，且剪枝成功率随 RVIS 频率增加而线性下降。
*   **动态优于静态**：DSTP 在仅保留 33.3% Token 的情况下，性能显著优于保留 66.6% Token 的静态剪枝方法，证明了“在正确的时间提供正确的视觉信息”比单纯增加 Token 数量更重要。
*   **高效性**：DSTP 引入的额外计算开销极小（TFLOPs 增加可忽略不计），且由于减少了模型因视觉信息缺失而产生的“胡言乱语（冗余生成）”，在某些场景下总推理延迟反而更低。

### 7. 优点（亮点）
*   **视角新颖**：首次从解码动态性的角度解释了剪枝在推理任务中失效的原因。
*   **即插即用**：无需重新训练模型，能够直接增强现有的任何视觉剪枝算法。
*   **逻辑自洽**：通过大量的注意力热图可视化直观地证明了 RVIS 的存在及其对推理的影响。
*   **性能卓越**：在 MathVerse 等高难度推理任务上，相比基准剪枝方法实现了大幅度的性能回升（如在 Qwen3 上从 32% 提升至 52%）。

### 8. 不足与局限
*   **层级一致性限制**：目前 DSTP 在所有 Transformer 层中统一应用相同的视觉 Token，未考虑不同层对视觉特征（低级 vs 高级）的差异化需求。
*   **超参数依赖**：阈值 $\tau$ 和时长 $L$ 目前是静态设置的，虽然具有一定的鲁棒性，但针对不同难度的查询，如果能实现自适应调整会更理想。
*   **极端压缩挑战**：在极高压缩率（如低于 10%）下，虽然 DSTP 有提升，但仍难以完全弥补信息损失。
