## 摘要
Token 剪枝已成为一种有效的方法，通过丢弃信息量较少的视觉 Token 并保持性能，来降低大规模视觉语言模型（LVLMs）巨大的计算开销。然而，现有方法通常依赖于来自不同 LVLM 组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为了解决这一问题，我们提出了 DeSAP，一种新颖的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间细粒度的跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的共同引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现稳健的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确性和效率方面均持续优于现有最先进（SOTA）方法。在 LLaVA-1.5-7B 上，DeSAP 仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

*   **现有挑战**：目前的 Token 剪枝方法主要分为“视觉中心”（依赖视觉编码器的 [CLS] 注意力）和“任务中心”（依赖 LLM 的跨模态注意力）。
*   **研究动机**：作者发现现有方法存在**注意力偏差**：视觉编码器的注意力往往偏向全局语义而忽略局部细节；而 LLM 侧的跨模态注意力在早期层存在严重的“空间偏差”（倾向于关注图像底部靠近文本的 Token）。
*   **核心目标**：探索如何在视觉编码器端（更早阶段）引入任务感知信息，以实现更精确、无偏差的 Token 剪枝。

### 2. 论文提出的方法论：DeSAP
作者提出了 **DeSAP（Decoupled Similarity-Aware Pruning）**，其核心思想是在视觉编码器内部通过解耦特征来实现任务引导。

*   **特征解耦（Decomposition）**：将 ViT 最后一层的隐藏状态分解为“残差分量”（捕获全局语义）和“解耦注意力分量”（保留局部细节）。实验证明，解耦后的注意力分量与文本的对齐度更高。
*   **细粒度跨模态对齐（FCMA）**：利用解耦后的视觉注意力特征与文本 Embedding 计算余弦相似度。为了减少噪声，引入了 Token 级别的权重贡献机制，计算出“解耦相似度”作为任务感知信号。
*   **双源引导剪枝（Dual-Source Guidance）**：
    *   **任务感知信号**：基于解耦相似度选择与查询最相关的 Token。
    *   **视觉显著信号**：基于 [CLS] 注意力选择图像中视觉上最显著的 Token。
    *   两者以 **50%:50%** 的比例分配预算，确保既不丢失任务目标，也不丢失背景上下文。
*   **Token 合并（Token Merging）**：对于未被选中的 Token，采用聚类方式将其信息合并到保留的 Token 中，以最小化信息损失。

*   **高性能与高压缩比**：在 LLaVA-1.5-7B 上，仅保留 11.1% 的 Token（64个），仍能保持 98.1% 的原始性能，并实现 10 倍的 FLOPs 降低。

*   **解决偏差问题**：定性分析显示，DeSAP 能精准定位到文本提到的细小物体，而传统方法往往因全局偏差而将其滤除。

### 7. 优点（亮点）
*   **训练无关（Training-free）**：作为一种即插即用的方法，不需要额外的微调成本。
*   **编码器端对齐**：首次在视觉编码器端实现细粒度的任务感知对齐，比在 LLM 侧剪枝更高效且能避免 LLM 的空间注意力偏差。
*   **鲁棒性强**：双源引导机制平衡了“任务相关性”与“视觉完整性”，使其在开放式问答和受限问答中表现均衡。

### 8. 不足与局限
*   **对模糊查询的依赖**：当文本查询非常简短或模糊（如“描述这张图”）时，任务感知信号的引导作用会减弱，模型会退化为依赖视觉显著性。

*   **应用限制**：主要针对 Transformer 架构的视觉编码器，对于非 Transformer 架构的适配性尚未讨论。
