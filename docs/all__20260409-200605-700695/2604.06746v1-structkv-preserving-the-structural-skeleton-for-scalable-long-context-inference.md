## 摘要
随着大语言模型（LLMs）扩展到支持超过一百万个 token 的上下文窗口，键值（KV）缓存的线性增长带来了严重的内存容量和带宽瓶颈，限制了长上下文推理的效率。现有的压缩方法通常根据局部显著性指标对 token 进行优先级排序，以将预填充计算与解码内存解耦。然而，这些方法往往依赖于特定层的局部显著性快照，从而系统性地丢弃了那些在整个网络深度中充当全局信息枢纽、但在选定的剪枝层中暂时处于休眠状态的 token。为了解决这一局限性，我们提出了 StructKV，这是一个结构感知的 KV 缓存压缩框架，它引入了三项核心创新：首先，全局入度中心性（Global In-Degree Centrality）聚合了整个网络深度的注意力模式，以识别全局信息枢纽；其次，动态枢轴检测（Dynamic Pivot Detection）利用信息论指标自适应地定位压缩的最佳层；最后，结构传播与解耦（Structural Propagation and Decoupling）将计算预算与内存存储预算分离。在 LongBench 和 RULER 基准测试上的实验结果表明，StructKV 有效地保留了长程依赖性和检索鲁棒性。

## Abstract
As Large Language Models (LLMs) scale to support context windows exceeding one million tokens, the linear growth of Key-Value (KV) cache imposes severe memory capacity and bandwidth bottlenecks, constraining the efficiency of long-context inference. Existing compression approaches typically prioritize tokens based on local saliency metrics to decouple prefill computation from decoding memory. However, these methods often rely on local saliency snapshots at a specific layer, thereby systematically discarding tokens that act as global information hubs across the network depth but appear temporarily dormant at the specific layer selected for pruning. To address this limitation, we propose StructKV, a structure-aware KV cache compression framework that introduces three core innovations: First, Global In-Degree Centrality aggregates attention patterns across the network depth to identify global information hubs. Second, Dynamic Pivot Detection utilizes information-theoretic metrics to adaptively locate the optimal layer for compression. Finally, Structural Propagation and Decoupling separates the computational budget from the memory storage budget. Experimental results on the LongBench and RULER benchmarks demonstrate that StructKV effectively preserves long-range dependencies and retrieval robustness.

随着大语言模型（LLM）支持的上下文窗口突破百万级别，**KV Cache（键值缓存）** 的线性增长成为了推理效率的致命瓶颈：
*   **预填充阶段（Prefill）**：计算复杂度随序列长度呈平方级增长，导致高延迟。

**研究动机**：现有的压缩方法（如 SnapKV, FastKV）通常基于特定层的“局部显著性快照”来决定保留哪些 token。作者发现这种方法具有“近视性”，容易误删那些在全局深度中起关键作用、但在特定层表现“休眠”的 token。StructKV 的核心目标是识别并保留上下文的**结构骨架（Structural Skeleton）**，以实现更稳健的长文本推理。

### 2. 论文提出的方法论
StructKV 引入了三个核心创新技术来构建结构感知的压缩框架：

*   **全局入度中心性（Global In-Degree Centrality, $C_{global}$）**：
    *   不再依赖单层注意力分数，而是通过递归累加从输入层到“枢轴层（Pivot Layer）”的所有注意力权重。
    *   引入层级衰减因子 $\lambda$，优先考虑深层语义信息。这种方法能识别出在多层中持续作为信息汇聚点的“全局枢纽” token。
*   **动态枢轴检测（Dynamic Pivot Detection）**：
    *   传统方法固定在某一层进行剪枝，而 StructKV 引入在线检测机制。
    *   通过监控**注意力熵（Entropy）**、**稀疏度（Sparsity）**和**方差（Variance）**的梯度变化，自动捕捉模型从“广泛探索”转向“聚焦提取”的相位转换点（Phase Transition），从而确定最佳剪枝层 $L^*$。
*   **结构传播与解耦（Structural Propagation & Decoupling）**：
    *   **计算与存储分离**：在枢轴层之后，后续深层仅对筛选出的“结构骨架”进行计算（降低预填充计算量），但独立维护一个更精简的 KV Cache 用于解码（优化显存）。
    *   通过重新对齐 Position ID 确保旋转位置编码（RoPE）的一致性。

*   **高效解耦**：成功解决了预填充速度、解码延迟和长文本准确率之间的“三难困境”。
