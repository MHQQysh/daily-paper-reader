## 摘要
对于多模态大语言模型（MLLMs），视觉信息相对于文本而言较为稀疏。因此，针对高效推理的视觉剪枝研究应运而生。目前的方法通常根据视觉编码器或 LLM 解码器中的注意力分数来衡量 Token 的重要性，进而保留高注意力分数的视觉 Token 并剪除其余部分。在本文中，我们提出了一种不同且更精准的方法。我们不依赖于特定机制的信号，而是在视觉和文本特征交互之前，直接计算两者之间的互信息（MI）。这使我们能够在特征层面显式地衡量跨模态依赖性。我们的 MI-Pruner 简单、高效且具有非侵入性，无需访问内部注意力图或修改模型架构。实验结果表明，我们的方法在极低延迟的情况下优于以往基于注意力的剪枝方法。

## Abstract
For multimodal large language models (MLLMs), visual information is relatively sparse compared with text. As a result, research on visual pruning emerges for efficient inference. Current approaches typically measure token importance based on the attention scores in the visual encoder or in the LLM decoder, then select visual tokens with high attention scores while pruning others. In this paper, we pursue a different and more surgical approach. Instead of relying on mechanism-specific signals, we directly compute Mutual Information (MI) between visual and textual features themselves, prior to their interaction. This allows us to explicitly measure crossmodal dependency at the feature levels. Our MI-Pruner is simple, efficient and non-intrusive, requiring no access to internal attention maps or architectural modifications. Experimental results demonstrate that our approach outperforms previous attention-based pruning methods with minimal latency.

#### 1. 核心问题与整体含义
多模态大语言模型（MLLM）在处理高分辨率图像时，视觉 Token 的数量远超文本，且由于自注意力机制的平方复杂度，导致推理时的计算开销和内存占用巨大。现有的视觉剪枝方法主要依赖视觉编码器或 LLM 解码器内部的**注意力分数（Attention Scores）**。然而，注意力分数存在位置偏差、注意力汇聚（Attention Sinks）等噪声，且提取注意力矩阵会带来额外的存储和计算延迟。

本文提出了 **MI-Pruner**，旨在通过信息论的方法，在视觉和文本特征进入 LLM 交互之前的**投影空间（Projection Space）**中，直接计算两者的**互信息（Mutual Information, MI）**，从而精准、高效地识别并保留与查询（Query）最相关的视觉 Token。

#### 2. 方法论
MI-Pruner 的核心思想是将 Token 剪枝建模为一个子集选择问题，通过最大化跨模态互信息来筛选 Token。

*   **核心步骤**：
    1.  **特征提取与归一化**：在投影层之后提取视觉嵌入 $V$ 和文本嵌入 $T$，并映射到单位超球面上。
    2.  **分布构建**：基于玻尔兹曼分布计算视觉-文本（跨模态）和视觉-视觉（模态内）的相似度矩阵，进而导出条件概率分布 $p(t|v)$ 和 $p(v|v)$。
    3.  **互信息量化**：利用**点向互信息（PMI）**衡量 Token 间的相关性。通过最大聚合（Maximal Aggregation）捕获最强的语义对齐信号。
    4.  **贪心搜索选择**：定义评分函数 $S_i = \lambda S_{cross} - (1-\lambda) S_{self}$。其中 $\lambda$ 用于平衡“跨模态相关性”与“模态内冗余性”。
    5.  **高效推理**：该方法具有非侵入性，不需要修改模型架构或访问内部注意力图，计算复杂度仅为 $O(N_V)$ 级别。

#### 6. 主要结论与发现
*   **性能领先**：MI-Pruner 在多个 Benchmark 上均达到了 SOTA 性能，尤其在极高剪枝率下（如仅保留 32 个 Token），其性能下降远小于基于注意力的方法。
*   **幻觉抑制**：在 POPE 评测中，MI-Pruner 能够更好地保留查询相关的物体信息，从而缓解模型在剪枝后容易出现的物体幻觉问题。
*   **高效性**：相比提取注意力图，直接在投影空间计算 MI 具有更低的计算开销和内存占用，且与 FlashAttention 等加速算子完全兼容。

#### 7. 优点
*   **理论严谨**：基于信息论中的互信息和子集选择理论，比启发式的注意力分数更具解释性。
*   **非侵入性与通用性**：作为插件模块，无需重新训练模型，可无缝集成到任何“编码器-投影层-解码器”架构的 MLLM 中。
*   **鲁棒性**：能够自适应地根据文本查询定位视觉区域，避免了注意力机制中常见的“注意力汇聚”噪声干扰。

#### 8. 不足与局限
*   **先验假设简化**：假设视觉 Token 的先验分布是均匀的，这忽略了摄影构图（如三分法、黄金分割）中不同位置重要性不同的先验知识。
*   **文本剪枝缺失**：目前仅针对视觉 Token 进行剪枝，未考虑长文本 Prompt 中的冗余 Token 压缩。
*   **动态预算**：目前仍需手动设定剪枝比例（Budget），未能实现根据输入内容复杂度自动决定保留 Token 数量的完全动态剪枝。
