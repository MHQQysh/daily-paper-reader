## 摘要
长上下文语言建模通常被视为 Token 级注意力的可扩展性挑战，但在现有方法中，从局部到全局的信息结构化在很大程度上仍是隐式的。借鉴话语理解的认知理论，我们提出了 HiCI（层级化构建-整合），这是一种层级化注意力模块，它构建分段级表示，将其整合到共享的全局上下文中，并广播两者以约束分段级注意力。我们通过对 LLaMA-2 进行参数高效的适配来验证 HiCI，仅增加了不到 5.5% 的额外参数，将上下文长度从 4K 扩展到 100K Token（7B 模型）和 64K Token（13B 模型）。在语言建模、检索和指令遵循基准测试中，HiCI 相比强基线模型取得了持续的改进，包括在主题检索上达到与闭源模型相当的水平，并在代码理解上超越了 GPT-3.5-Turbo-16K。这些结果证明了显式层级结构化作为长上下文建模归纳偏置的有效性。

## Abstract
Long-context language modeling is commonly framed as a scalability challenge of token-level attention, yet local-to-global information structuring remains largely implicit in existing approaches. Drawing on cognitive theories of discourse comprehension, we propose HiCI (Hierarchical Construction--Integration), a hierarchical attention module that constructs segment-level representations, integrates them into a shared global context, and broadcasts both to condition segment-level attention. We validate HiCI through parameter-efficient adaptation of LLaMA-2 with only <5.5% additional parameters, extending context from 4K to 100K tokens (7B) and 64K tokens (13B). Across language modeling, retrieval, and instruction-following benchmarks, HiCI yields consistent improvements over strong baselines, including matching proprietary models on topic retrieval and surpassing GPT-3.5-Turbo-16K on code comprehension. These results demonstrate the effectiveness of explicit hierarchical structuring as an inductive bias for long-context modeling.

### 1. 核心问题与研究动机
*   **核心问题**：大型语言模型（LLMs）在处理长文本时面临**效率**（注意力机制的平方复杂度）与**有效性**（难以可靠建模长程依赖）的双重挑战。
*   **研究动机**：现有的长文本扩展技术（如位置插值、稀疏注意力、循环架构）大多关注 Token 级别的计算优化，缺乏显式的从局部到全局的信息结构化处理。
*   **理论背景**：受认知科学中“构建-整合模型”（Construction-Integration）和“全局工作空间理论”（Global Workspace Theory）启发，认为人类理解文本是一个层级化过程：先构建局部表示，再整合为全局连贯的认知，最后通过全局信息引导后续处理。

### 2. 方法论：HiCI 架构
HiCI 模块将注意力计算分为三个显式阶段，作为 Transformer 的层级化归纳偏置：
1.  **局部构建 (Local Construction)**：
    *   将输入序列划分为固定长度的段落（Segments）。
    *   使用一组可学习的查询槽（Query Slots）通过交叉注意力从每个段落中提取压缩的局部表示 $L_i$。
2.  **全局整合 (Global Integration)**：
    *   对所有段落的局部表示进行多视图统计池化（计算均值、最大/最小值、标准差、归一化均值）。
    *   通过共享的瓶颈压缩层和注意力权重，将这些统计特征整合为紧凑的全局上下文 $G$。
3.  **自顶向下广播 (Top-down Broadcast)**：
    *   将全局上下文 $G$ 和段落特定的局部表示 $L_i$ 拼接在每个段落的 Key-Value 序列前。
    *   段落内的 Token 在进行注意力计算时，会同时关注到这些层级化背景信息，实现全局信息的并行注入。

### 3. 实验设计
*   **基础模型**：LLaMA-2 (7B 和 13B)。
*   **数据集与基准测试**：
    *   **语言建模**：PG-19 和 Proof-pile（评估困惑度 PPL）。
    *   **检索任务**：Passkey Retrieval（密钥检索）和 Topic Retrieval（长聊天主题检索）。
    *   **下游指令遵循**：LongBench（包含单/多文档问答、摘要、Few-shot、合成任务、代码等 21 个子任务）。
*   **对比方法**：LongLoRA（主要基线）、GPT-3.5-Turbo-16K、Claude-1.3、LongChat-13B 等。

### 4. 资源与算力
*   **硬件环境**：使用了 **8 张 NVIDIA H100 (80GB) GPU**。
*   **训练配置**：
    *   采用 LoRA 微调，仅训练 HiCI 模块、LoRA 适配器、Embedding 和 Norm 层。
    *   持续预训练阶段：在 RedPajama 数据集上训练 1000 步，有效 Batch Size 为 64。
    *   指令微调阶段：在 LongAlpaca-12k 上训练 5 个 Epoch。
*   **效率表现**：在 100K 长度训练时，由于 HiCI 允许更细粒度的分段，其训练速度比 LongLoRA 快约 19.3%，尽管内存开销略高（增加 3.5%~9.9%）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 8K 到 100K 的多种训练长度，以及从 2K 到 100K 的评估长度。
*   **消融实验**：非常充分。作者验证了全局整合（G）和局部构建（L）的各自贡献、表示容量（槽位数量 $M, K$）的影响，以及段落粒度（Segment Size）的敏感性。
*   **客观性**：实验不仅对比了开源模型，还对比了闭源商业模型（GPT-4/3.5），并使用了标准的第三方基准测试（LongBench），评估过程较为客观公平。

### 6. 主要结论与发现
*   **性能提升**：HiCI 在所有评估长度下的 PPL 均优于 LongLoRA；在 LongBench 综合得分上比 LongLoRA 高出 2.6%。
*   **检索能力强**：在主题检索中，HiCI-13B 在 13K 长度内达到 100% 准确率，匹配了闭源模型水平；在 Passkey 任务中表现出比基线更强的长度外推稳健性。
*   **代码理解突出**：在 LongBench 的代码类别中，HiCI 超过了 GPT-3.5-Turbo-16K 达 9.7%。
*   **层级路由涌现**：分析发现，随着模型层数加深，注意力会越来越倾向于关注全局槽位（深层关注度是浅层的 3-5 倍），证明了层级化信息流的有效性。

### 7. 优点与亮点
*   **认知启发**：将心理学/认知科学理论成功转化为神经网络架构，提供了明确的归纳偏置。
*   **参数高效**：仅需增加约 5.5% 的额外参数即可实现显著的上下文扩展。
*   **长度不变性**：在训练一致的评估下，HiCI 表现出近乎“长度无关”的 PPL 稳定性，而传统方法随长度增加波动较大。
*   **推理灵活**：HiCI 模块在推理时是可选的，既可以用于加速 Prefill 阶段，也可以切换回全注意力模式。

### 8. 不足与局限
*   **参数开销**：虽然比例不高，但相比纯位置插值方法，HiCI 引入了额外的可学习参数和模块复杂度。
*   **分段依赖**：实验表明 HiCI 对段落大小（Segment Size）较为敏感，较小的段落有利于层级整合但可能影响局部感受野，需仔细调优。
*   **训练成本**：尽管比全量微调便宜，但仍需要 H100 等高端算力进行持续预训练才能发挥最佳效果。
*   **任务偏差**：在某些合成任务（Synthetic Tasks）上，HiCI 的提升并不明显，可能说明该架构更适合具有自然语言结构的长文本，而非无结构的随机序列。
