## 摘要
Transformer 缺乏一种显式的架构机制来存储和组织训练过程中获取的知识。我们引入了可学习的稀疏记忆库：一组随机初始化并端到端训练的潜代币（latent tokens），Transformer 层通过交叉注意力机制对其进行查询，以检索存储的知识。为了在不产生过高注意力开销的情况下扩展记忆容量，我们受专家混合（Mixture-of-Experts）架构启发，提出了基于章节的路由机制，将记忆库划分为若干章节，并训练一个路由器为每个输入选择相关的子集。这使得在保持计算可控的同时，能够扩展到 262K 个记忆代币。我们在预训练和指令微调的相关基准测试中，将该方法与标准 Transformer（在等效 FLOPs 设置下）进行了评估。我们的模型超越了等效 FLOPs 的基准模型，这表明存在一个新的扩展维度，并证明了显式关联记忆提供了与模型参数中隐式捕获的知识互补的能力。此外，我们观察到在持续训练过程中知识保留能力有所提高，并且在不同训练阶段转换（例如从预训练到指令微调）时具有抗遗忘的鲁棒性。

## Abstract
Transformers lack an explicit architectural mechanism for storing and organizing knowledge acquired during training. We introduce learnable sparse memory banks: a set of latent tokens, randomly initialized and trained end-to-end, that transformer layers query via cross-attention to retrieve stored knowledge. To scale memory capacity without prohibitive attention costs, we propose chapter-based routing inspired by Mixture-of-Experts architectures, partitioning the memory bank into chapters and training a router to select relevant subsets per input. This enables scaling to 262K memory tokens while maintaining tractable computation. We evaluate our approach against standard transformers (in iso-FLOP settings) on pre-training and instruction fine-tuning across relevant benchmarks. Our models surpass iso-FLOP baselines suggesting scope for a new axis of scaling, demonstrating that explicit associative memory provides complementary capacity to what is captured implicitly in model parameters. Additionally, we observe improved knowledge retention under continued training, with robustness to forgetting when transitioning between training phases (e.g., pretraining to instruction fine-tuning).

### 1. 核心问题与研究动机
Transformer 模型虽然强大，但其知识存储主要隐式地分布在稠密参数中，缺乏一种显式的、可独立扩展的架构机制来存储和组织训练中获取的事实性知识。现有的长文本处理技术（如 KV 缓存压缩、RAG）多侧重于推理时的上下文管理，而非模型内部的学习记忆。
**核心动机：** 探索一种可学习的、显式的、且能在大规模容量下保持计算高效的内部存储机制，作为模型参数化知识的补充。

### 2. 方法论：Mixture of Chapters (MoC)
*   **核心思想：** 在 Transformer 块中引入一个由可学习潜在 Token（Latent Tokens）组成的“记忆库”（Memory Bank）。模型通过交叉注意力（Cross-Attention）查询这些 Token。
*   **关键技术细节：**
    *   **记忆层结构：** 记忆库 $M$ 是一个随机初始化并随模型端到端训练的矩阵。Token 流作为 Query，记忆库提供 Key 和 Value。
    *   **章节路由（Chapter-based Routing）：** 为了避免对海量记忆 Token 进行全量注意力的昂贵开销，作者借鉴 MoE（混合专家）思想，将记忆库划分为 $C$ 个“章节”（Chapters）。
    *   **序列级路由：** 采用轻量级路由器，通过对输入序列状态进行池化（Pooling）得到表示向量，计算各章节的得分，并仅选择 Top-k 个章节参与当前的交叉注意力计算。
    *   **计算效率：** 将复杂度从 $O(L \cdot N_m)$ 降低到 $O(L \cdot k \cdot T)$（其中 $L$ 是序列长度，$N_m$ 是总记忆量，$k$ 是选定章节数，$T$ 是每章 Token 数）。

### 3. 实验设计
*   **数据集：** 预训练使用 FineWeb-Edu（10B Token 子集）；指令微调（IFT）使用 UltraChat_200k。
*   **Benchmark：** 评估了 MMLU（综合知识）、ARC-Challenge（科学推理）、BoolQ（布尔问答）和 OpenBookQA。
*   **对比方法：**
    1.  **Vanilla (backbone-only)：** 移除记忆模块的原始 16 层 Transformer。
    2.  **Vanilla (iso-FLOP)：** 增加层数（至 24 层）以匹配 MoC 模型预训练算力的稠密 Transformer 基准。
    3.  **Mixture of Chapters (MoC)：** 本文提出的 16 层骨干 + 4 个记忆层的模型。

### 4. 资源与算力
*   **硬件：** 预训练使用 8 张 GPU 进行 DDP 训练；指令微调使用 4 张 GPU。

*   **参数量：** 稠密基准约 203M 参数；MoC 模型总参数约 371M（含 201M 的记忆库），但由于稀疏路由，推理/训练时的激活参数量与基准相当。

*   **实验覆盖：** 涵盖了预训练验证集损失、下游任务准确率、指令微调后的知识保留能力。
*   **消融实验：** 针对指令微调阶段是否冻结记忆库、学习率大小对性能的影响进行了对比。
*   **充分性评价：** 实验设计较为客观，特别是引入了 **iso-FLOP 基准**，排除了“性能提升仅源于计算量增加”的干扰。但在模型绝对规模（仅约 200M-400M 参数）和训练数据量（10B Token）上仍属于小规模验证，尚未在大模型（7B+）上验证扩展性。

### 6. 主要结论与发现
*   **性能领先：** 在相同算力预算下，MoC 模型的预训练损失低于稠密基准，且在知识密集型任务（如 MMLU, BoolQ）上表现更好。
*   **抗遗忘性：** 在经历高强度的指令微调后，稠密模型在 ARC-C 和 BoolQ 上出现了严重的性能退化（灾难性遗忘），而 MoC 模型表现极其稳定。
*   **记忆锚定：** 实验发现，在微调阶段即使冻结记忆库，模型性能依然稳健，说明预训练习得的显式记忆能起到“知识锚点”的作用。

### 7. 优点与亮点
*   **新扩展维度：** 提出了一种除了增加层数和隐藏维度之外的新扩展路径——扩展显式记忆容量。
*   **稀疏高效：** 成功将 MoE 的路由思想引入记忆检索，实现了 262K Token 的超大记忆库。
*   **模块化：** 记忆库可以独立于骨干参数进行更新或冻结，为知识编辑和持续学习提供了可能。

### 8. 不足与局限
*   **规模限制：** 实验主要在亿级参数模型上进行，缺乏在百亿或千亿级参数模型上的验证。
*   **路由粒度：** 目前采用序列级路由，可能导致同一序列内不同 Token 对记忆的需求无法精细化满足（未来可探索 Token 级路由）。
*   **评估范围：** 尚未测试长文本生成、复杂逻辑推理等更广泛的 LLM 能力。
*   **VRAM 占用：** 虽然计算量（FLOPs）对齐了，但显式记忆库占用了额外的显存空间。
