## 摘要
语言模型难以泛化到预训练上下文长度之外，这限制了长程推理和检索能力。在长上下文数据上进行持续预训练会有所帮助，但由于注意力机制（Attention）的二次方缩放特性，其成本非常高昂。我们观察到，大多数 token 并不需要对整个序列进行（全局）注意力计算，而是可以依赖局部上下文。基于此，我们提出了 L2A（Learning To Attend），这是一种通过决定何时调用全局注意力来实现条件化（逐 token）长程记忆访问的层。我们在 Qwen 2.5 和 Qwen 3 模型上评估了 L2A，将其有效上下文长度从 32K 扩展到了 128K token。L2A 在跳过约 80% token 的全局注意力计算的同时，其性能与标准长上下文训练的差距在 3% 以内，优于之前的基准模型。我们还设计了定制的 Triton 内核，以便在 GPU 上高效实现这种逐 token 的条件注意力，与 FlashAttention 相比，训练吞吐量和首字延迟（time-to-first-token）提升了高达约 2 倍。此外，L2A 支持对高度稀疏的全局注意力层进行训练后剪枝，在性能损失微乎其微的情况下，将 KV 缓存内存占用减少了高达 50%。

## Abstract
Language models struggle to generalize beyond pretraining context lengths, limiting long-horizon reasoning and retrieval. Continued pretraining on long-context data can help but is expensive due to the quadratic scaling of Attention. We observe that most tokens do not require (Global) Attention over the entire sequence and can rely on local context. Based on this, we propose L2A (Learning To Attend), a layer that enables conditional (token-wise) long-range memory access by deciding when to invoke global attention. We evaluate L2A on Qwen 2.5 and Qwen 3 models, extending their effective context length from 32K to 128K tokens. L2A matches the performance of standard long-context training to within 3% while skipping Global Attention for $\sim$80% of tokens, outperforming prior baselines. We also design custom Triton kernels to efficiently implement this token-wise conditional Attention on GPUs, achieving up to $\sim$2x improvements in training throughput and time-to-first-token over FlashAttention. Moreover, L2A enables post-training pruning of highly sparse Global Attention layers, reducing KV cache memory by up to 50% with negligible performance loss.

*   **核心问题**：大语言模型（LLMs）在处理长文本时面临计算开销随长度呈二次方增长的瓶颈。现有的长上下文扩展方法（如持续预训练）成本极高。

*   **整体含义**：论文旨在探索一种**逐 token 的条件式记忆访问机制**，让模型自主学习“何时”需要访问长程记忆，从而在不损失性能的前提下大幅降低长文本处理的计算和内存开销。

### 2. 方法论：核心思想与关键技术
*   **核心思想（L2A 层）**：将标准的注意力层替换为 L2A 层，该层由**局部注意力（Local Attention）**、**路由决策器（Router）**和**全局注意力（Global Attention）**组成。
*   **关键技术细节**：
    *   **两阶段处理**：所有 token 首先通过滑动窗口注意力（SWA）获取局部表示。
    *   **动态路由**：路由模块（线性层+Sigmoid）根据局部表示计算得分。若得分超过阈值（如 0.5），则触发昂贵的全局注意力；否则跳过。
    *   **训练策略**：使用标准下次词预测（NTP）损失，并加入**稀疏正则化项**（惩罚全局注意力的调用）。为了防止路由塌陷（即模型为了省事全部走局部路径），引入了随机采样机制，在训练时以小概率强制所有 token 调用全局注意力以获取梯度。
    *   **Triton 内核优化**：开发了定制的 GPU 内核，通过紧凑缓存（Compacted Buffer）和索引映射，在硬件层面实现了对稀疏 token 的高效加速，避免了 GPU 在处理不规则稀疏计算时的性能损失。

    *   **长上下文评估**：使用 HELMET（包含合成检索、RAG、长文本 ICL 等）、BabiLong（事实链推理）和 MRCR（多轮指代消解）。

### 5. 实验数量与充分性
*   **实验规模**：论文涵盖了从 1.5B 到 8B 不同参数规模的模型，并在 8K 到 128K 的多个长度梯度上进行了测试。
*   **消融实验**：非常充分。包括：
    1.  **路由有效性**：对比了“上下文无关”的随机稀疏路由，证明了模型确实学到了根据内容决策。
    2.  **窗口大小影响**：测试了从 0 到 4K 的局部窗口，发现模型能自动调节稀疏度来补偿窗口缩减。
    3.  **正则化系数 $\lambda$**：分析了稀疏度与性能的权衡。
    4.  **层剪枝**：验证了训练后直接丢弃 50% 极稀疏层的可行性。
*   **客观性**：对比了多种主流的稀疏注意力变体，并提供了内核层面的速度对比，实验设计较为严谨、客观。

### 6. 主要结论与发现
*   **性能持平**：L2A 在仅使用约 20% 全局注意力的情况下，性能与全量注意力的差距保持在 1.5%–3% 以内。
*   **效率翻倍**：训练吞吐量和首字延迟（TTFT）均提升了约 2 倍。
*   **KV 缓存压缩**：通过训练后剪枝（Pruning），可以在几乎不损失性能的情况下减少 **50% 的 KV 缓存内存占用**。
*   **任务差异性**：发现不同任务对全局注意力的需求不同，ICL（上下文学习）最需要全局信息，而合成检索任务则非常稀疏。

### 7. 优点
*   **自适应性**：不同于以往固定稀疏模式的方法，L2A 实现了“按需分配”计算资源。
*   **软硬件协同**：不仅提出了算法，还通过 Triton 内核解决了稀疏计算在 GPU 上“不跑分”的顽疾。
*   **易于集成**：作为标准注意力层的掉入式替换（Drop-in replacement），可以直接用于现有模型的微调。

### 8. 不足与局限
*   **KV 缓存仍需全量存储（除非剪枝）**：虽然计算减少了，但为了支持未来 token 可能的全局访问，默认仍需存储所有历史 KV，这对推理内存的压力依然存在（论文通过层剪枝部分缓解了此问题）。
*   **训练稳定性**：条件计算（Conditional Computation）容易导致训练不稳定或路由失效，虽然论文提出了采样策略，但在更大规模（如 70B+）模型上的稳定性尚未验证。
*   **任务覆盖**：主要集中在理解和检索类任务，对于极长文本的创造性生成（如写长篇小说）的逻辑连贯性探讨较少。
