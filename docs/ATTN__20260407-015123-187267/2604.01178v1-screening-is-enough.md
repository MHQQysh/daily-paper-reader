## 摘要
标准 softmax 注意力的一个核心局限在于它没有定义绝对的查询-键（query-key）相关性概念：注意力权重是通过根据相对得分在所有键之间重新分配固定的单位质量来获得的。因此，相关性仅相对于竞争键来定义，且无法显式拒绝无关的键。我们引入了 Multiscreen，这是一种围绕我们称之为“筛选”（screening）的机制构建的语言模型架构，该机制实现了绝对的查询-键相关性。筛选机制不再在所有键之间重新分配注意力，而是根据显式阈值评估每个键，丢弃无关键并聚合剩余键，从而消除了键之间的全局竞争。在各项实验中，Multiscreen 以比 Transformer 基准模型少约 40% 的参数实现了相当的验证损失，能够在显著更大的学习率下进行稳定优化，在长上下文困惑度方面保持了强劲性能，即使在远超训练上下文长度的情况下，检索性能也几乎没有下降，并在 100K 上下文长度下将推理延迟降低了高达 3.2 倍。

## Abstract
A core limitation of standard softmax attention is that it does not define a notion of absolute query--key relevance: attention weights are obtained by redistributing a fixed unit mass across all keys according to their relative scores. As a result, relevance is defined only relative to competing keys, and irrelevant keys cannot be explicitly rejected. We introduce Multiscreen, a language-model architecture built around a mechanism we call screening, which enables absolute query--key relevance. Instead of redistributing attention across all keys, screening evaluates each key against an explicit threshold, discarding irrelevant keys and aggregating the remaining keys, thereby removing global competition among keys. Across experiments, Multiscreen achieves comparable validation loss with approximately 40% fewer parameters than a Transformer baseline, enables stable optimization at substantially larger learning rates, maintains strong performance in long-context perplexity, shows little to no degradation in retrieval performance even far beyond the training context length, and reduces inference latency by up to 3.2$\times$ at 100K context length.

### 1. 核心问题与研究动机
*   **Softmax 注意力的局限性**：标准 Transformer 使用的 Softmax 注意力是一种“相对竞争”机制。它将固定的单位质量（总和为 1）分配给所有 Key。这意味着一个 Key 获得高权重仅仅是因为它比其他 Key 强，而不是因为它本身“足够相关”。
*   **无关信息的干扰**：在长文本中，即使所有 Key 都与 Query 无关，Softmax 也会强制分配权重，导致注意力被稀释（Attention Fading）。
*   **绝对相关性的缺失**：模型缺乏一种显式拒绝（Reject）无关信息并定义“绝对相关性”的手段。
*   **研究目标**：提出一种名为 **Multiscreen** 的新架构，通过“筛选（Screening）”机制替代传统的注意力分配，实现更高效的长文本处理、更强的检索能力和更好的参数效率。

### 2. 核心方法论
Multiscreen 架构的核心是**门控筛选分片（Gated Screening Tile）**，其关键技术细节包括：
*   **筛选单元（Screening Unit）**：
    *   **单位长度归一化**：将 Query、Key、Value 全部归一化到单位超球面上，使相似度（点积）严格限制在 $[-1, 1]$ 之间。
    *   **Trim-and-Square 变换**：引入显式阈值。只有相似度超过阈值的 Key 才会获得非零相关性得分，且得分随相似度增加平滑上升。
    *   **软掩码（Softmask）与学习窗口**：每个筛选单元学习一个动态窗口大小 $w$。超出窗口的 Key 被硬剔除，窗口边缘通过余弦函数平滑过渡。
    *   **TanhNorm 聚合**：由于相关性得分不再求和为 1，聚合后的向量范数可能波动。使用 TanhNorm 函数在保留方向的同时，平滑地将输出范数限制在 1 以内。
*   **微型位置编码（MiPE）**：一种仅在学习窗口较小时激活的旋转位置编码（类似 RoPE）。当模型需要长程关注时，MiPE 自动失效，避免了长文本外推时的位置偏差。
*   **门控机制**：结合了类似 GLU 的乘法门控，将筛选后的上下文信息与输入特征进行非线性融合。

### 6. 主要结论与发现
*   **参数效率**：在相同验证损失下，Multiscreen 比 Transformer 节省约 **40% 的参数**。
*   **训练稳定性**：Multiscreen 对超大学习率具有极强的耐受性（在 Transformer 崩溃的速率下仍能稳定收敛），这归功于其消除了 Key 之间的竞争。
*   **长文本检索**：在远超训练长度（如 $2^{17}$）的情况下，Multiscreen 的检索准确率几乎不下降，而 Transformer 即使使用了最佳的外推技术也会迅速崩溃。
*   **推理加速**：在 100K 上下文时，推理速度比 Transformer 快 **2.3 到 3.2 倍**，且随着长文本微调，加速效果更明显（因为更多窗口保持有限大小）。

### 7. 优点与亮点
*   **架构创新**：从根本上挑战了“注意力必须归一化”的假设，提出了“筛选”这一更符合逻辑的长文本处理方式。
*   **外推能力**：通过学习窗口和 MiPE 的设计，巧妙解决了位置编码在长文本下的泛化难题。
*   **实用性强**：不仅在理论指标（Loss）上占优，在实际推理延迟和检索准确率上也有巨大提升。

### 8. 不足与局限
*   **超大规模验证不足**：虽然展示了 4B 模型的潜力，但由于算力限制，该模型处于欠训练状态，尚未在 70B+ 等主流大模型规模上验证其 Scaling Law 的持续性。

*   **实现复杂度**：筛选机制需要自定义 Triton 内核才能发挥最大效能，相比成熟的 FlashAttention 生态，其部署门槛稍高。
