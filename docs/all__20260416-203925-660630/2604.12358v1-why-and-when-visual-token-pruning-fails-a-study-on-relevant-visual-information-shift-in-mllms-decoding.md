## 摘要
近年来，视觉 Token 剪枝已被用于处理多模态大语言模型（MLLMs）中海量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但在有效泛化到复杂的视觉推理任务时却面临困难，这是以往研究中尚未充分探讨的关键差距。通过系统性分析，我们将解码过程中的“相关视觉信息偏移”（Relevant Visual Information Shift, RVIS）确定为导致失效的主要原因。为解决这一问题，我们提出了“解码阶段偏移感知 Token 剪枝”（Decoding-stage Shift-aware Token Pruning, DSTP），这是一种无需训练的附加框架，能够使现有的剪枝方法在解码阶段将视觉 Token 与不断变化的推理需求相对齐。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能下降，同时在视觉理解基准测试中也一致地带来了性能提升。此外，DSTP 在多种先进架构上均表现出有效性，突显了其泛化能力以及极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：为了降低 MLLM 处理大量视觉标记（Visual Tokens）带来的计算和内存开销，现有的研究多采用“视觉标记剪枝”技术。
*   **核心问题**：作者发现，现有的剪枝方法（如 FastV、DivPrune 等）在简单的视觉理解（如 VQA）中表现良好，但在复杂的**视觉推理（Visual Reasoning）**任务（如数学竞赛、逻辑谜题）中性能会剧烈下降。
*   **根本原因**：论文首次提出了**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**的概念。在复杂推理中，模型在解码的不同阶段需要关注图像的不同区域，而现有的静态剪枝方法在“预填充（Prefill）”阶段就永久丢弃了后续步骤可能需要的标记。

### 2. 方法论：DSTP 框架
作者提出了一种名为 **DSTP（Decoding-stage Shift-aware Token Pruning）** 的无需训练的插件式框架，包含两个核心模块：
*   **RISD（相关视觉信息偏移检测）**：
    *   在解码过程中，实时监控当前生成的 Token 对视觉标记的注意力分布。
    *   计算当前注意力向量与预填充阶段锚点注意力的余弦相似度。
    *   当相似度低于设定阈值 $\tau$ 时，判定发生了 RVIS，触发标记更新。
*   **CPTS（上下文保留视觉标记交换）**：
    *   **动态重评估**：重新计算所有视觉标记（包括之前被丢弃的）的重要性得分。
    *   **上下文保留交换**：不直接替换，而是将新识别的重要标记与原始预填充阶段保留的标记取**并集**，形成一个临时的扩展标记集。
    *   **持续时间控制**：该扩展集仅维持 $L$ 个解码步骤，随后恢复原状，以平衡推理精度与计算效率。

### 6. 主要结论与发现
*   **RVIS 是失效主因**：复杂推理任务中 RVIS 发生频率远高于简单任务，静态剪枝无法应对这种动态的需求偏移。
*   **DSTP 的有效性**：在保留相同比例标记的情况下，DSTP 显著提升了推理任务的准确率（例如在 MathVerse 上将 FastV 的性能从 32.23% 提升至 52.54%）。
*   **效率优势**：DSTP 在 33.3% 的标记保留率下，性能甚至超过了保留 66.6% 标记的静态剪枝方法，且计算成本更低。

### 7. 优点（亮点）
*   **洞察深刻**：首次揭示了 MLLM 解码过程中视觉关注点的动态偏移特性（RVIS）。
*   **即插即用**：无需重新训练模型，能够直接集成到现有的各种剪枝算法中。
*   **上下文保留设计**：通过“并集”操作而非简单替换，保证了模型对图像全局理解的连续性，避免了生成崩溃。

### 8. 不足与局限
*   **超参数依赖**：阈值 $\tau$ 和时长 $L$ 目前仍需手动设定，虽然论文给出了推荐值，但不同模型或任务可能存在最优解差异。
*   **层级一致性限制**：目前 DSTP 在所有 Transformer 层中统一应用相同的视觉标记，未考虑不同层对视觉信息需求的差异（如浅层关注细节，深层关注语义）。
*   **极端压缩挑战**：在极高的剪枝率下，虽然 DSTP 有所改善，但仍难以完全达到原始全量标记模型的水平。
