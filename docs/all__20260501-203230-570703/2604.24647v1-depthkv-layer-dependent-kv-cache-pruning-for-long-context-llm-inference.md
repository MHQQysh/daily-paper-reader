## Abstract
Long-context reasoning is a critical capability of large language models (LLMs), enabling applications such as long-document understanding, summarization, and code generation. However, efficient autoregressive inference relies on the key-value (KV) cache, whose memory footprint grows linearly with sequence length, leading to a major memory bottleneck. To mitigate this overhead, KV cache pruning methods discard cached tokens with low attention scores during inference. Most existing methods apply a uniform pruning ratio across layers, implicitly assuming that all layers contribute equally to overall model performance. We show that this assumption is suboptimal, as layers differ significantly in their sensitivity to pruning. We propose DepthKV, a layer-dependent pruning framework that allocates a fixed global KV budget across layers based on their sensitivity, rather than using a uniform allocation. Across multiple models and tasks, DepthKV consistently outperforms uniform pruning at the same global pruning ratio, demonstrating more effective utilization of the KV cache budget through layer-dependent allocation.

### 1. 核心问题与研究动机
*   **背景**：大语言模型（LLM）在处理长文本时，KV Cache（键值缓存）的内存占用随序列长度线性增长，成为显存瓶颈。
*   **核心问题**：现有的 KV Cache 剪枝方法（如 H2O, SnapKV 等）通常假设所有 Transformer 层同等重要，因此在各层应用**均匀的剪枝比例**。
*   **研究动机**：作者通过实验发现，Transformer 的不同层对剪枝的敏感度存在显著差异。均匀分配预算会导致关键层的信息丢失，从而限制了模型在极低内存预算下的表现。

### 2. 方法论：DepthKV 框架
*   **核心思想**：放弃均匀分配，根据各层的重要性（敏感度）非均匀地分配固定的全局 KV 缓存预算。
*   **关键技术细节**：
    1.  **层敏感度分析**：通过逐层剪枝的消融实验，识别出对性能影响最大的层。
    2.  **内容放大层（Content Amplification Layers）**：研究发现某些特定层负责维持生成的丰富度和长度，剪枝这些层会导致输出枯竭（YapScore 显著下降）。
    3.  **表示度量指标（InfoNCE）**：引入对比学习损失 InfoNCE 作为代理指标。实验证明，InfoNCE 越低的层（表示越鲁棒），在剪枝时性能下降越剧烈，因此应分配更多预算。
*   **分配策略**：
    *   **MLP (Middle-Layer Protection)**：保护中间层，对其他层进行均匀剪枝。
    *   **MGA (Metric-Guided Allocation)**：利用 InfoNCE 指标自动为各层计算剪枝比例，实现异构分配。
    *   **MLMA (Middle-Layer Metric Allocation)**：结合上述两者的混合策略。

### 6. 主要结论与发现
1.  **层重要性不均**：Transformer 层对 KV 剪枝的敏感度呈非均匀分布，中间层通常比早期或晚期层更关键。
2.  **指标有效性**：InfoNCE 是层重要性的可靠预测器，能够指导预算分配而无需昂贵的消融搜索。
3.  **性能优势**：DepthKV 在所有任务中均优于均匀剪枝基准，尤其在需要复杂推理和长程依赖的任务中提升显著。
4.  **内容维持**：保护“内容放大层”能有效防止模型在长文本生成时出现过早停止或输出无意义片段的问题。

### 7. 优点
*   **突破常规**：挑战了 KV 剪枝中“均匀分配”的默认假设，提出了更精细化的资源管理视角。
*   **通用性强**：作为一种启发式、非查询感知的后训练方法，它可以轻松集成到现有的 KV 压缩技术（如 H2O 或 SnapKV）之上。
*   **理论支撑**：通过表示学习中的几何和鲁棒性指标（如 InfoNCE）为层重要性提供了理论解释。

### 8. 不足与局限
*   **非查询感知（Non-query-aware）**：剪枝决策在预填充阶段一次性完成，无法根据解码过程中不断变化的 Query 动态找回已丢弃的 Token。
*   **粒度限制**：目前仅在“层”级别进行优化，未考虑同一层内不同“注意力头（Heads）”的敏感度差异。
*   **静态分配**：虽然是层依赖的，但分配方案在推理开始后是静态的，可能无法完美适配极长文本中发生的主题偏移。
