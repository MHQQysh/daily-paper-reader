## 摘要
多模态大语言模型（MLLMs）由于视觉标记（token）序列的高度冗余，面临着巨大的计算开销。现有方法通常采用单层视觉 Transformer (ViT) 特征和静态剪枝策略来解决这一问题。然而，这种固定配置在面对多样化的指令时往往表现得较为脆弱。为了克服这些局限性，我们提出了 CLASP，这是一个基于类别自适应层融合和双阶段剪枝的即插即用标记缩减框架。具体而言，CLASP 首先通过多层视觉特征融合构建特定类别的视觉表示。随后，它执行双阶段剪枝，在用于保证相关性的注意力显著枢轴标记（pivot tokens）和用于保证覆盖范围的冗余感知补充标记（completion tokens）之间分配标记预算。通过类别自适应剪枝，CLASP 实现了基于提示词条件的特征融合和预算分配，从而实现了激进且鲁棒的视觉标记缩减。广泛的实验表明，CLASP 在各种基准测试、剪枝比例和 MLLM 架构中均一致优于现有方法。代码将在 https://github.com/Yunkaidang/CLASP 发布。

## Abstract
Multimodal Large Language Models (MLLMs) suffer from substantial computational overhead due to the high redundancy in visual token sequences. Existing approaches typically address this issue using single-layer Vision Transformer (ViT) features and static pruning strategies. However, such fixed configurations are often brittle under diverse instructions. To overcome these limitations, we propose CLASP, a plug-and-play token reduction framework based on class-adaptive layer fusion and dual-stage pruning. Specifically, CLASP first constructs category-specific visual representations through multi-layer vision feature fusion. It then performs dual-stage pruning, allocating the token budget between attention-salient pivot tokens for relevance and redundancy-aware completion tokens for coverage. Through class-adaptive pruning, CLASP enables prompt-conditioned feature fusion and budget allocation, allowing aggressive yet robust visual token reduction. Extensive experiments demonstrate that CLASP consistently outperforms existing methods across a wide range of benchmarks, pruning ratios, and MLLM architectures. Code will be available at https://github.com/Yunkaidang/CLASP.

多模态大语言模型（MLLMs）在处理高分辨率图像时，视觉标记（Token）数量呈平方级增长（如从 576 增加到 2880 个），导致巨大的计算开销、内存占用和推理延迟。
*   **现有局限**：目前的 Token 缩减方法大多采用**静态策略**，即固定使用 Vision Transformer (ViT) 的某一层特征，并应用统一的剪枝比例。
*   **研究动机**：作者发现不同的指令对视觉抽象层次的需求不同（例如：计数/OCR 需要浅层细节，而场景理解需要深层语义）。因此，固定的特征提取和剪枝策略在多样化的任务面前表现脆弱。

#### 2. 论文提出的方法论
CLASP 提出了一种**即插即用**的框架，核心思想是根据输入指令的类别动态调整视觉特征的融合方式和 Token 的剪枝逻辑。

*   **类别自适应层融合 (Class-Adaptive Layer Fusion)**：
    *   **路由机制**：使用轻量级文本路由器将用户提示词（Prompt）映射到预定义的意图类别（如对象识别、文本识别、空间关系等）。
    *   **动态融合**：根据类别权重，对 ViT 不同层的特征进行凸组合（Convex Combination）。这使得模型能针对细节任务保留浅层特征，针对语义任务利用深层特征。
*   **类别自适应双阶段剪枝 (Class-Adaptive Dual-Stage Pruning)**：
    *   **阶段一：相关性保留（Attention Pivots）**：基于跨模态注意力分数，筛选出与指令最相关的 $K_1$ 个“枢轴 Token”，确保关键证据不丢失。
    *   **阶段二：覆盖范围补充（Coverage Completion）**：在剩余 Token 中，利用**冗余感知聚类（Spherical K-Means）**筛选出 $K_2$ 个 Token。该阶段旨在最小化与已选枢轴的冗余度，确保视觉背景的全局覆盖。
    *   **动态预算分配**：根据指令类别动态调整 $K_1$ 和 $K_2$ 的比例（例如：复杂推理任务分配更多预算给覆盖范围，细节任务分配给相关性）。
