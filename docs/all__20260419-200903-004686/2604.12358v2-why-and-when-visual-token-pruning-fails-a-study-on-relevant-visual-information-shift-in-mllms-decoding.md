## 摘要
近期，视觉 Token 剪枝已被广泛研究，以处理多模态大语言模型（MLLMs）中海量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但在复杂视觉推理任务中却难以有效泛化，这是以往研究中尚未充分探讨的关键空白。通过系统性分析，我们将解码过程中的“相关视觉信息偏移”（Relevant Visual Information Shift, RVIS）确定为导致失效的主要原因。为解决这一问题，我们提出了“解码阶段偏移感知 Token 剪枝”（Decoding-stage Shift-aware Token Pruning, DSTP），这是一个无需训练的插件式框架，使现有的剪枝方法能够在解码阶段根据不断变化的推理需求对齐视觉 Token。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能下降，同时在视觉理解基准测试中也持续带来了性能提升。此外，DSTP 在多种先进架构上均表现出有效性，突显了其泛化能力以及极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：现有的视觉 Token 剪枝方法（如 FastV, DivPrune）在简单的视觉问答（VQA）任务中表现良好，但在复杂的视觉数学推理（VMR）任务中性能大幅下降。
*   **核心发现**：作者通过分析发现，复杂推理是一个多步骤过程，模型在解码的不同阶段需要关注图像的不同区域。这种现象被定义为**相关视觉信息偏移（Relevant Visual Information Shift, RVIS）**。
*   **核心问题**：传统的剪枝方法通常在“预填充（Prefill）”阶段一次性决定保留哪些 Token，这种静态策略无法适应解码过程中动态变化的视觉需求，导致推理所需的关键信息被永久丢弃。

### 2. 方法论：DSTP 框架
作者提出了一种无需训练的插件式框架 **DSTP（Decoding-stage Shift-aware Token Pruning）**，其核心思想是在解码阶段动态监测并调整视觉 Token。

*   **RISD 模块（相关视觉信息偏移检测）**：
    *   在解码的每一步，计算当前生成的 Token 对视觉 Token 的注意力分布。
    *   通过计算当前注意力向量与预填充阶段“锚点”注意力向量之间的**余弦相似度**来衡量偏移程度。
    *   当相似度低于设定阈值 $\tau$ 时，触发 RVIS 预警。
*   **CPTS 模块（上下文保留视觉 Token 交换）**：
    *   **动态重评估**：一旦检测到 RVIS，模型会重新评估所有（包括之前被剪枝掉的）视觉 Token 的重要性。
    *   **上下文保留交换**：为了防止生成中断或逻辑混乱，DSTP 不直接替换 Token，而是将“初始保留的 Token”与“新识别的相关 Token”取**并集**，形成一个临时的扩展 Token 集。
    *   **持续时间控制**：这种扩展状态会维持 $L$ 个解码步，随后恢复到初始剪枝状态，以平衡性能与效率。

### 6. 主要结论与发现
*   **RVIS 是失效主因**：推理任务中 RVIS 发生频率极高，而简单理解任务中注意力相对固定。
*   **静态剪枝的局限**：静态剪枝在 RVIS 发生时成功率骤降，证明了“一次性剪枝”不适用于长链推理。
*   **DSTP 的有效性**：DSTP 显著提升了推理任务的性能（例如在 MathVerse 上将 FastV 的性能从 32% 提升至 52%），且对简单任务也有微小增益。
*   **效率平衡**：DSTP 引入的额外计算开销极小（TFLOPs 增加不到 1%），甚至在某些情况下因减少了无效推理步骤而降低了端到端延迟。

### 7. 优点（亮点）
*   **视角新颖**：首次系统性地识别并量化了 MLLM 解码过程中的视觉信息偏移现象。
*   **即插即用**：无需重新训练模型，能够直接提升现有多种剪枝方法的鲁棒性。
*   **逻辑自洽**：通过“并集”操作保留上下文，解决了动态更换 Token 可能导致的模型“幻觉”或生成中断问题。

### 8. 不足与局限
*   **超参数依赖**：阈值 $\tau$ 和持续步数 $L$ 目前是静态设置的，针对不同难度或类型的题目，动态调整这些参数可能会更优。
*   **层级一致性**：目前 DSTP 在所有 Transformer 层统一应用相同的 Token 集，未考虑不同层对视觉细节需求的差异（早期层关注低级特征，深层关注高级语义）。
*   **极端长文本**：对于极长序列的推理，频繁触发 Token 交换可能会累积一定的 KV Cache 管理开销。
