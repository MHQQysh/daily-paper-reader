## 摘要
大语言模型（LLM）智能体在动态环境中难以自主演化出协作策略，这主要是因为粗粒度的全局结果掩盖了局部策略优化所需的因果信号。我们将这一瓶颈识别为多智能体信用分配问题，该问题在经典多智能体强化学习（MARL）中已有长期研究，但在基于 LLM 的系统中仍未得到充分解决。基于这一观察，我们提出了 LangMARL 框架，将协作式 MARL 中的信用分配和策略梯度演化引入语言空间。LangMARL 引入了智能体级语言信用分配，开创了语言空间中的策略改进梯度演化，并从回放轨迹中总结出与任务相关的因果关系，以提供稠密反馈并改善稀疏奖励下的收敛性。在多种协作式多智能体任务上的广泛实验表明，该方法提高了样本效率、可解释性，并具有很强的泛化能力。

## Abstract
Large language model (LLM) agents struggle to autonomously evolve coordination strategies in dynamic environments, largely because coarse global outcomes obscure the causal signals needed for local policy refinement. We identify this bottleneck as a multi-agent credit assignment problem, which has long been studied in classical multi-agent reinforcement learning (MARL) but remains underaddressed in LLM-based systems. Building on this observation, we propose LangMARL, a framework that brings credit assignment and policy gradient evolution from cooperative MARL into the language space. LangMARL introduces agent-level language credit assignment, pioneers gradient evolution in language space for policy improvement, and summarizes task-relevant causal relations from replayed trajectories to provide dense feedback and improve convergence under sparse rewards. Extensive experiments across diverse cooperative multi-agent tasks demonstrate improved sample efficiency, interpretability, and strong generalization.

### 2. 方法论：LangMARL 框架
LangMARL 模拟了经典 MARL 的“中心化训练，去中心化执行”（CTDE）范式，但在语言空间内操作：
*   **语言策略执行器（Language Policy Actor）**：每个智能体持有一个以自然语言描述的策略（包含指令、规则、示例），根据局部观测生成动作。
*   **中心化语言评论家（Centralized Language Critic）**：在训练阶段，评论家获取完整的全局轨迹，进行因果推理，将团队表现分解为针对每个智能体的**语言信用（Language Credit）**，明确指出谁做得好、谁导致了瓶颈。
*   **语言策略梯度估计器（Language Policy Gradient Estimator）**：将语言信用转化为“语言梯度”——即具体的策略修改建议，指明策略应如何调整。
*   **语言策略优化器（Language Policy Optimizer）**：类似于 Adam 等优化器，它语义化地聚合多个轨迹产生的语言梯度，解决冲突并更新智能体的自然语言策略。

*   **理论迁移创新**：成功将 MARL 的数学严谨性（信用分配、梯度更新）与 LLM 的语义理解能力结合。
*   **高可解释性**：由于策略和反馈均以自然语言形式存在，人类可以清晰地理解智能体为何改进策略以及改进了什么。
*   **易用性工具包**：提供了类似 TorchRL 的 API 设计，降低了开发者构建自进化多智能体系统的门槛。

### 8. 不足与局限
*   **长程任务挑战**：在极长步骤（如数百步）的任务中，稀疏奖励和误差累积仍是挑战，因果链条过长可能导致信用分配失效。
*   **拓扑结构固定**：目前实验基于预定义的智能体拓扑，尚未实现动态增减智能体或动态改变组织结构。
*   **成本问题**：频繁的轨迹回放、评论家分析和策略优化涉及大量的 LLM API 调用，在大规模应用时可能面临较高的 Token 成本。
