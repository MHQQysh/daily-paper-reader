## 摘要
Token 剪枝已成为降低大语言视觉模型（LVLMs）巨大计算开销的一种有效方法，它通过丢弃信息量较少的视觉 Token，同时保持模型性能。然而，现有方法通常依赖于来自 LVLM 不同组件的单一注意力源，由于注意力分布存在偏差，导致剪枝决策不完整且并非最优。为解决这一问题，我们提出了 DeSAP，一种新颖的解耦相似度感知剪枝方法，用于在视觉编码器内进行精确的任务感知 Token 剪枝。具体而言，DeSAP 引入了解耦相似度来捕捉视觉特征与文本 Token 之间细粒度的跨模态相关性，为剪枝提供明确的任务相关引导。通过将解耦相似度与源自视觉注意力的视觉显著性信号相结合，DeSAP 在任务相关线索和视觉线索的双重引导下进行 Token 剪枝，即使在激进的剪枝比例下也能实现稳健的剪枝。在不同基准测试和架构上的广泛实验表明，DeSAP 在准确率和效率方面均持续优于现有最先进（SOTA）方法。在 LLaVA-1.5-7B 上，DeSAP 仅保留 11.1% 的视觉 Token，实现了 10 倍的 FLOPs 减少和 2.3 倍的预填充加速，同时保持了 98.1% 的原始性能。

## Abstract
Token pruning has emerged as an effective approach to reduce the substantial computational overhead of Large Vision-Language Models (LVLMs) by discarding less informative visual tokens while preserving performance. However, existing methods typically rely on individual attention sources from different LVLM components, resulting in incomplete and suboptimal pruning decisions due to biased attention distributions. To address this problem, we propose DeSAP, a novel Decoupled Similarity-Aware Pruning method for precise, task-aware token pruning within the visual encoder. Specifically, DeSAP introduces a decoupled similarity to capture fine-grained cross-modal relevance between visual features and text tokens, providing explicit task-related guidance for pruning. By integrating decoupled similarity with visual saliency signals derived from visual attention, DeSAP performs token pruning under the guidance of both task-related and visual cues, enabling robust pruning even under aggressive pruning ratios. Extensive experiments across diverse benchmarks and architectures show that DeSAP consistently outperforms SOTA methods in both accuracy and efficiency. On LLaVA-1.5-7B, DeSAP achieves a 10 times FLOPs reduction and a 2.3 times prefill speedup by retaining only 11.1% of visual tokens, while maintaining 98.1% of the original performance.

### 论文总结：大语言视觉模型中基于解耦相似度的任务感知 Token 剪枝 (DeSAP)

#### 1. 核心问题与整体含义（研究动机和背景）
大型视觉语言模型（LVLMs）在处理高分辨率图像或多帧视频时，会产生海量的视觉 Token，导致巨大的计算开销和推理延迟。现有的 Token 剪枝方法主要分为两类：
*   **视觉中心化（Visual-centric）：** 依赖视觉编码器的注意力（如 [CLS] 注意力），但容易偏向全局语义而忽略局部细节。
*   **任务中心化（Task-centric）：** 在 LLM 侧利用跨模态注意力，但由于 LLM 的空间偏见（倾向于关注序列中邻近的底部 Token），容易导致早期信息丢失。
**核心问题：** 现有的注意力分布存在偏差，无法在视觉编码器端提供精确的任务感知引导。本文旨在探索如何更早地在编码器端引入任务相关信息，以实现更稳健的剪枝。

#### 2. 方法论：DeSAP 核心思想与技术细节
DeSAP（Decoupled Similarity-Aware Pruning）的核心思想是在视觉编码器内部通过“解耦”操作提取细粒度的任务引导信号。
*   **隐藏状态分解：** 将视觉编码器最后一层的特征分解为**解耦残差组件**（捕获全局语义）和**解耦注意力组件**（保留局部细节）。实验发现，去除残差连接后的注意力特征能更好地与文本对齐。
*   **细粒度跨模态对齐（FCMA）：** 计算解耦后的视觉注意力特征与文本嵌入之间的余弦相似度。为了减少噪声，引入了 Token 级别的权重贡献机制，通过衡量 Token 与对方模态全局代表性特征（如 [CLS] 或 [EOS]）的对齐程度来加权。
*   **双源引导剪枝：** 结合“任务相关相似度图”和“视觉显著性图（[CLS] 注意力）”，两者各分配 50% 的保留配额。这种双重引导确保了模型既能关注任务目标，又不会丢失重要的背景上下文。
*   **Token 合并：** 采用基于聚类的合并策略，将未选中的 Token 聚合到最相似的保留 Token 中，以最大程度减少信息损失。

*   **基准模型（Backbones）：** LLaVA-1.5-7B, LLaVA-NeXT-7B, Video-LLaVA-7B, 以及最新的 Qwen-2.5-VL-7B。

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 4 种不同架构的模型上进行了验证，覆盖了从图像到视频的多种任务。
*   **剪枝比例：** 测试了从 66.7% 到 95% 的多种剪枝比例，特别是在 88.9% 的激进剪枝率下进行了深度对比。
*   **消融实验：** 针对任务/视觉保留比例分配、解耦编码策略（$qqv, qkv$ 等）、以及细粒度对齐策略（FCMA）的有效性进行了详细的消融研究。
*   **评价：** 实验设计非常充分且客观，通过跨模型、跨任务的对比证明了方法的通用性和稳健性。

#### 6. 主要结论与发现
*   **性能卓越：** 在 LLaVA-1.5-7B 上，仅保留 11.1% 的 Token（64个），DeSAP 仍能保持 98.1% 的原始性能，平均准确率显著超过其他 SOTA 方法（如比 HoloV 高出 2.3%）。
*   **效率提升：** 实现了 10 倍的 FLOPs 减少，在预填充阶段（Prefill）实现了 2.3 倍的加速。
*   **解耦价值：** 证明了视觉编码器的隐藏状态分解是获取细粒度任务引导的关键，克服了传统注意力机制的全局偏见。

#### 7. 优点（亮点）
*   **训练无关（Training-free）：** 这是一个即插即用的方法，不需要对模型进行任何微调。
*   **细粒度对齐：** 首次在编码器端实现了高效的细粒度跨模态对齐，比在 LLM 侧进行剪枝更具前瞻性。
*   **小目标保护：** 可视化结果显示，DeSAP 在处理图像中仅占极小比例的任务相关物体（小目标）时，比现有方法更精准。

#### 8. 不足与局限
*   **对 Query 的依赖：** 当文本查询（Query）语义模糊或过于简短（如 Caption 任务）时，该方法会退化并更多地依赖 [CLS] 注意力。
*   **长文本开销：** 虽然引入了文本 Token 筛选策略，但极长的文本输入仍可能增加相似度计算的负担。
*   **动态比例：** 目前任务引导和视觉引导的比例是固定分配（50/50），未来若能根据任务类型动态调整比例可能会进一步提升效果。
