## 摘要
近年来，视觉 Token 剪枝已被用于处理多模态大语言模型（MLLMs）中海量的视觉 Token。然而，我们观察到，虽然现有的剪枝方法在简单的视觉理解任务上表现可靠，但在复杂视觉推理任务上的泛化效果较差，这是以往研究中尚未充分探讨的关键差距。通过系统性分析，我们将解码过程中的“相关视觉信息偏移”（Relevant Visual Information Shift, RVIS）确定为导致失效的主要原因。为解决这一问题，我们提出了“解码阶段偏移感知 Token 剪枝”（Decoding-stage Shift-aware Token Pruning, DSTP），这是一种无需训练的插件式框架，能够使现有的剪枝方法在解码阶段将视觉 Token 与不断变化的推理需求相对齐。大量实验表明，DSTP 显著缓解了剪枝方法在复杂推理任务中的性能下降，同时在视觉理解基准测试中也一致地提升了性能。此外，DSTP 在多种先进架构上均表现出有效性，突显了其泛化能力以及极低计算开销下的高效性。

## Abstract
Recently, visual token pruning has been studied to handle the vast number of visual tokens in Multimodal Large Language Models. However, we observe that while existing pruning methods perform reliably on simple visual understanding, they struggle to effectively generalize to complex visual reasoning tasks, a critical gap underexplored in previous studies. Through a systematic analysis, we identify Relevant Visual Information Shift (RVIS) during decoding as the primary failure driver. To address this, we propose Decoding-stage Shift-aware Token Pruning (DSTP), a training-free add-on framework that enables existing pruning methods to align visual tokens with shifting reasoning requirements during the decoding stage. Extensive experiments demonstrate that DSTP significantly mitigates performance degradation of pruning methods in complex reasoning tasks, while consistently yielding performance gains even across visual understanding benchmarks. Furthermore, DSTP demonstrates effectiveness across diverse state-of-the-art architectures, highlighting its generalizability and efficiency with minimal computational overhead.

### 1. 核心问题与整体含义（研究动机和背景）
*   **背景**：MLLM 处理高分辨率图像时会产生海量的视觉 Token，导致推理时的计算和内存开销巨大。现有的视觉 Token 剪枝方法（如 FastV）旨在通过剔除冗余 Token 来加速推理。
*   **核心问题**：作者发现现有的剪枝方法在简单的视觉问答（VQA）任务中表现良好，但在复杂的**视觉推理（VMR）**任务（如数学竞赛、逻辑谜题）中性能会剧烈下降。
*   **研究动机**：探究为何静态的剪枝策略在推理任务中失效。研究发现，推理是一个多步过程，模型在不同阶段需要的视觉信息是动态变化的，而传统方法在预填充（Prefill）阶段一次性剔除 Token 的做法无法适应这种动态需求。

### 2. 方法论：DSTP 框架
作者提出了 **DSTP (Decoding-stage Shift-aware Token Pruning)**，这是一个无需训练的插件式框架，核心思想是“**监测偏移，动态找回**”。
*   **RISD (相关视觉信息偏移检测)**：
    *   在解码的每一步，计算当前生成的 Token 对视觉区域的注意力分布。
    *   通过计算当前注意力向量与预填充阶段初始注意力向量的**余弦相似度**来监测偏移。
    *   当相似度低于设定阈值 $\tau$ 时，判定发生了**相关视觉信息偏移（RVIS）**。
*   **CPTS (上下文保留视觉 Token 交换)**：
    *   一旦检测到 RVIS，系统会重新评估所有原始视觉 Token（包括之前被剪枝掉的）。
    *   **动态交换**：选出当前最相关的 Top-K 个 Token。
    *   **上下文保留**：不直接替换，而是将新选出的 Token 与初始保留的 Token 取**并集**，形成一个临时的扩展集合，以维持推理的连贯性。
    *   **恢复机制**：该扩展状态持续 $L$ 个解码步后，自动恢复到初始剪枝状态，以平衡计算效率。

### 6. 主要结论与发现
*   **RVIS 是失效主因**：复杂推理任务中，视觉焦点会随逻辑步频繁移动（RVIS 频率高），而简单任务焦点相对固定。
*   **DSTP 效果显著**：在保留 33.3% Token 的情况下，DSTP 将剪枝模型的推理性能从原始剪枝方法的 ~50% 恢复到了接近 Vanilla 模型的 ~90% 水平。
*   **效率优势**：DSTP 引入的计算开销（TFLOPs）增加不到 1%，且由于它能减少模型因信息缺失而产生的冗余重复生成，在某些情况下总推理延迟反而比原始剪枝方法更低。

### 7. 优点
*   **理论贡献**：首次系统性地定义并量化了解码阶段的视觉信息偏移（RVIS）现象。
*   **无需训练**：作为一种免训练（Training-free）方法，它可以直接应用于任何现有的 MLLM 和剪枝算法，具有极强的通用性。
*   **设计巧妙**：通过“并集交换”和“限时恢复”策略，巧妙地解决了推理一致性与计算效率之间的矛盾。

### 8. 不足与局限
*   **层级策略缺失**：目前 DSTP 在所有 Transformer 层中统一应用相同的视觉 Token 集合，未考虑不同层（如浅层感知 vs. 深层推理）对视觉信息需求的差异。
*   **超参数敏感性**：阈值 $\tau$ 和持续步数 $L$ 仍需手动调优，虽然论文给出了推荐值，但在极端不同的任务中可能需要重新设定。
*   **自适应性潜力**：目前触发机制是基于固定阈值，未来若能结合强化学习实现自动触发，效果可能更佳。
