## 摘要
视频大语言模型 (VideoLLMs) 在视频理解方面展现了卓越的能力，但海量的输入视频 token 给部署带来了巨大的计算负担。现有方法主要在输入层面对视频 token 进行剪枝，而忽略了视频和大语言模型 (LLMs) 中固有的信息结构。针对这一问题，我们提出了 HieraVid，这是一个分层剪枝框架，能够逐步且动态地减少视觉冗余。基于视频具有“片段-帧”结构以及 LLM 内部单向传播多模态信息这两个观察结果，我们将剪枝分解为三个层面：1) 片段级，首先对视频 token 进行时间分段和空间合并；2) 帧级，对同一片段内的相似帧进行联合剪枝以保持多样性；3) 层级，随着 LLM 层数的增加，冗余在不损害性能的情况下逐渐减少。我们在四个广泛使用的视频理解基准测试上进行了大量实验，以全面评估 HieraVid 的有效性。值得注意的是，在仅保留 30% token 的情况下，HieraVid 达到了新的 SOTA 性能，同时分别保持了 LLaVA-Video-7B 和 LLaVA-OneVision-7B 98% 和 99% 以上的性能。

## Abstract
Video Large Language Models (VideoLLMs) have demonstrated impressive capabilities in video understanding, yet the massive number of input video tokens incurs a significant computational burden for deployment. Existing methods mainly prune video tokens at input level while neglecting the inherent information structure embedded in videos and large language models (LLMs). To address this, we propose HieraVid, a hierarchical pruning framework that progressively and dynamically reduces visual redundancy. Based on two observations that videos possess the segment-frame structure and LLMs internally propagate multi-modal information unidirectionally, we decompose pruning into three levels: 1) segment-level, where video tokens are first temporally segmented and spatially merged; 2) frame-level, where similar frames within the same segment are jointly pruned to preserve diversity; 3) layer-level, redundancy gradually shrinks as LLM layer increases w/o compromising performance. We conduct extensive experiments on four widely used video understanding benchmarks to comprehensively evaluate the effectiveness of HieraVid. Remarkably, with only 30% of tokens retained, HieraVid achieves new state-of-the-art performance, while maintaining over 98% and 99% of the performance of LLaVA-Video-7B and LLaVA-OneVision-7B, respectively.

#### 1. 论文的核心问题与整体含义
随着视频大语言模型（VideoLLMs）的发展，处理高分辨率和长视频的需求日益增长。然而，视频产生的海量视觉 Token 导致了巨大的计算负担，尤其是在具有平方复杂度的自注意力机制中，这显著增加了推理延迟和内存消耗。
**研究动机：** 现有的 Token 剪枝方法往往只在输入层进行简单处理，或者忽略了视频特有的“片段-帧”结构，以及大语言模型（LLM）内部多模态信息流动的方向性（通常向深层和指令 Token 聚集）。本文旨在提出一种无需训练的、能够充分利用视频结构和模型特性的高效剪枝框架。

#### 2. 方法论：核心思想与关键技术
HieraVid 提出了一个三层的分层剪枝框架，旨在动态且逐步地减少视觉冗余：

*   **片段级合并（Segment-level Merging）：**
    *   **合并率引导的分段：** 计算相邻帧之间对应位置 Token 的空间相似度，定义“重叠率”。当重叠率低于阈值 $\beta$ 时划定分段边界，将视频划分为语义连续的片段。
    *   **空间合并：** 在每个片段内，对时间维度上静态或变化极小的空间位置进行 Token 合并，消除空间冗余。
*   **帧级 DPP 剪枝（Frame-level DPP Pruning）：**
    *   **预算分配：** 根据片段的代表性和多样性，动态为不同片段分配剪枝预算。
    *   **多样性最大化：** 引入行列式点过程（DPP）算法，在片段内选择最具代表性且冗余度最低的帧。
    *   **指令感知：** 将视觉 Token 与最后一条指令 Token 的相关性得分融入 DPP 核矩阵，确保保留的 Token 既具多样性又与任务相关。
*   **层级平衡（Layer-level Balancing）：**
    *   **多阶段剪枝：** 基于 LLM 内部信息流向深层传播的观察，将 LLM 划分为多个阶段（如 4 个阶段），随着层数加深逐步增加剪枝比例，在浅层保留丰富特征，在深层精简冗余。

#### 7. 优点（亮点）
*   **结构感知：** 不同于以往随机或仅基于注意力的剪枝，HieraVid 深入挖掘了视频的“片段-帧”层级结构。

*   **即插即用：** 无需任何微调或额外训练，能够直接应用于现有的预训练 VideoLLMs。

#### 8. 不足与局限
*   **超参数依赖：** 框架涉及分段阈值 $\beta$、权重 $\lambda$ 等多个超参数，针对不同风格的视频（如极速切换的剪辑视频 vs. 静态监控视频）可能需要微调。
