## 摘要
长上下文推理是大语言模型（LLM）的一项关键能力，支持长文档理解、摘要生成和代码生成等应用。然而，高效的自回归推理依赖于键值（KV）缓存，其内存占用随序列长度线性增长，从而导致了主要的内存瓶颈。为了减轻这一开销，KV 缓存剪枝方法在推理过程中丢弃注意力分数较低的缓存令牌。大多数现有方法在各层之间应用统一的剪枝率，隐含地假设所有层对模型整体性能的贡献是均等的。我们证明了这一假设并非最优，因为不同层对剪枝的敏感度存在显著差异。我们提出了 DepthKV，这是一个分层剪枝框架，它根据各层的敏感度在层间分配固定的全局 KV 预算，而非采用统一分配。在多个模型和任务中，DepthKV 在相同的全局剪枝率下始终优于统一剪枝，证明了通过分层分配可以更有效地利用 KV 缓存预算。

## Abstract
Long-context reasoning is a critical capability of large language models (LLMs), enabling applications such as long-document understanding, summarization, and code generation. However, efficient autoregressive inference relies on the key-value (KV) cache, whose memory footprint grows linearly with sequence length, leading to a major memory bottleneck. To mitigate this overhead, KV cache pruning methods discard cached tokens with low attention scores during inference. Most existing methods apply a uniform pruning ratio across layers, implicitly assuming that all layers contribute equally to overall model performance. We show that this assumption is suboptimal, as layers differ significantly in their sensitivity to pruning. We propose DepthKV, a layer-dependent pruning framework that allocates a fixed global KV budget across layers based on their sensitivity, rather than using a uniform allocation. Across multiple models and tasks, DepthKV consistently outperforms uniform pruning at the same global pruning ratio, demonstrating more effective utilization of the KV cache budget through layer-dependent allocation.

### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）支持的上下文窗口不断扩大（从128K到数百万token），自回归推理中的**KV缓存（Key-Value Cache）**成为了主要的内存瓶颈。KV缓存随序列长度线性增长，极易超出GPU显存限制。
*   **现有问题**：目前的KV缓存剪枝方法（如H2O、SnapKV等）通常在所有Transformer层上应用**统一的剪枝比例**。
*   **研究动机**：作者发现不同层对剪枝的敏感度显著不同，中间层往往比早期或晚期层更关键。统一分配预算忽略了这种层间差异，导致资源利用效率低下。

### 2. 方法论：DepthKV 框架
DepthKV的核心思想是：**在固定全局内存预算的前提下，根据各层的“敏感度”非均匀地分配KV缓存配额。**

*   **层敏感度评估**：通过消融实验发现，剪枝某些层会导致性能剧烈下降，这些层被称为“内容放大层”（Content Amplification Layers）。
*   **重要性度量指标（InfoNCE）**：作者引入了表示学习中的InfoNCE度量，发现层表示的鲁棒性（对输入扰动的稳定性）与该层对剪枝的敏感度呈强负相关。
*   **分配策略**：
    *   **MGA（指标引导分配）**：利用InfoNCE分数计算各层的权重，给更敏感（重要）的层分配更多的KV缓存预算。
    *   **MLP（中间层保护）**：基于经验观察，直接保护中间层不被过度剪枝。
    *   **MLMA（混合策略）**：结合结构化保护和指标引导，保留特定中间层，其余预算按指标分配。
*   **令牌选择算法**：在确定每层预算后，结合H2O（基于注意力分数）或其改进的Value-aware（结合Value向量范数）方法来选择保留哪些具体的token。

### 6. 主要结论与发现
*   **层间差异显著**：Transformer各层对KV剪枝的敏感度存在统计学上的显著差异，统一剪枝策略确实是次优的。
*   **内容放大效应**：剪枝关键层不仅降低准确率，还会抑制模型生成内容的长度和信息量（YapScore下降）。

### 7. 优点
*   **即插即用**：作为一种后训练方法，无需微调，易于集成到现有推理框架中。
*   **资源高效**：通过精细化分配，在不增加额外内存开销的情况下提升了长文本处理质量。
*   **理论支撑**：结合了表示学习指标（InfoNCE）来解释层重要性，比纯启发式方法更具说服力。

### 8. 不足与局限
*   **非查询感知（Non-query-aware）**：剪枝决策在预填充阶段完成，无法根据解码过程中的动态查询（Query）实时调整保留的token。
*   **层内头差异**：目前只关注了层间的预算分配，未考虑同一层内不同注意力头（Heads）的重要性差异。
